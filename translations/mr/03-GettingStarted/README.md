## सुरुवात करणे  

[![Build Your First MCP Server](../../../translated_images/mr/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(या धड्यास的视频 पाहण्यासाठी वरच्या प्रतिमेला क्लिक करा)_

या विभागात अनेक धडे आहेत:

- **1 तुमचा पहिला सर्व्हर**, या पहिल्या धड्यात, तुम्हाला तुमचा पहिला सर्व्हर कसा तयार करायचा आणि इन्स्पेक्टर टूलद्वारे त्याची तपासणी कशी करायची हे शिकवले जाईल, तुम्हाला तुमच्या सर्व्हरची चाचणी आणि डीबगिंग करण्याचा मौल्यवान मार्ग, [धड्यासाठी येथे क्लिक करा](01-first-server/README.md)

- **2 क्लायंट**, या धड्यात, तुम्हाला क्लायंट कसा लिहायचा जो तुमच्या सर्व्हरशी कनेक्ट होऊ शकतो, शिकवले जाईल, [धड्यासाठी येथे क्लिक करा](02-client/README.md)

- **3 LLM सह क्लायंट**, क्लायंट लिहिण्याचा आणखी चांगला मार्ग म्हणजे त्यात LLM जोडणे जेणेकरून तो तुमच्या सर्व्हरशी काय करायचे यावर "वाटाघाट" करू शकेल, [धड्यासाठी येथे क्लिक करा](03-llm-client/README.md)

- **4 GitHub Copilot एजंट मोड वापरून सर्व्हर वापरणे Visual Studio Code मध्ये**. येथे, आपण Visual Studio Code मधून आमचा MCP सर्व्हर कसा चालवायचा हे पाहणार आहोत, [धड्यासाठी येथे क्लिक करा](04-vscode/README.md)

- **5 stdio ट्रान्सपोर्ट सर्व्हर** stdio ट्रान्सपोर्ट हे स्थानिक MCP सर्व्हर-टू-क्लायंट संवादासाठी शिफारस केलेले मानक आहे, सुरक्षित सबप्रॉसेस-आधारित संवाद प्रदान करते ज्यामध्ये अंगभूत प्रक्रिया पृथक्करण असते [धड्यासाठी येथे क्लिक करा](05-stdio-server/README.md)

- **6 MCP सह HTTP स्ट्रीमिंग (स्ट्रीम करण्याजोगा HTTP)**. आधुनिक HTTP स्ट्रीमिंग ट्रान्सपोर्टबद्दल शिका (हा दूरस्थ MCP सर्व्हरसाठी शिफारस केलेला दृष्टिकोन आहे [MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http) नुसार), प्रगती सूचना, आणि कसे स्केलेबल, रिअल-टाइम MCP सर्व्हर आणि क्लायंट्स स्ट्रीम करण्याजोग्या HTTP वापरून तयार करायचे. [धड्यासाठी येथे क्लिक करा](06-http-streaming/README.md)

- **7 VSCode साठी AI Toolkit वापरणे** जे वापरून तुम्ही तुमचे MCP क्लायंट्स आणि सर्व्हर चालवू आणि चाचणी करू शकता [धड्यासाठी येथे क्लिक करा](07-aitk/README.md)

- **8 चाचणी**. येथे आपण वेगवेगळ्या मार्गांनी आपल्या सर्व्हर आणि क्लायंटची चाचणी कशी करायची यावर विशेष लक्ष केंद्रित करू, [धड्यासाठी येथे क्लिक करा](08-testing/README.md)

- **9 तैनाती**. हा प्रकरण तुमच्या MCP सोल्यूशन्सची तैनाती करण्याच्या विविध मार्गांवर नजर टाकेल, [धड्यासाठी येथे क्लिक करा](09-deployment/README.md)

- **10 प्रगत सर्व्हर वापर**. हा प्रकरण प्रगत सर्व्हर वापर समाविष्ट करतो, [धड्यासाठी येथे क्लिक करा](./10-advanced/README.md)

- **11 प्रमाणीकरण**. हा प्रकरण सोपे प्रमाणीकरण कसे जोडायचे ते हाताळतो, बेसिक ऑथपासून JWT आणि RBAC वापरण्यासाठी. येथे सुरू करण्याचे प्रोत्साहन दिले जाते आणि नंतर प्रगत विषय पहा प्रकरण 5 मध्ये तसेच प्रकरण 2 मधील सुरक्षा वाढवण्यासाठी शिफारसी वापरा, [धड्यासाठी येथे क्लिक करा](./11-simple-auth/README.md)

- **12 MCP होस्ट्स**. लोकप्रिय MCP होस्ट क्लायंट्स कॉन्फिगर करा आणि वापरा ज्यात Claude Desktop, Cursor, Cline आणि Windsurf यांचा समावेश आहे. ट्रान्सपोर्ट प्रकार आणि समस्या निराकरण जाणून घ्या, [धड्यासाठी येथे क्लिक करा](./12-mcp-hosts/README.md)

- **13 MCP इन्स्पेक्टर**. MCP इन्स्पेक्टर टूल वापरून तुम्ही आपल्या MCP सर्व्हरचे इंटरऐक्टिव डीबग आणि चाचणी करा. समस्या निराकरणासाठी टूल्स, संसाधने, आणि प्रोटोकॉल संदेश शिकत जा, [धड्यासाठी येथे क्लिक करा](./13-mcp-inspector/README.md)

- **14 सॅम्पलिंग**. असे MCP सर्व्हर तयार करा जे LLM संबंधित कार्यांवर MCP क्लायंट्स सोबत सहकार्य करतात (`2026-07-28` रीलिझ उमेदवारामध्ये बंद केलेले; तरीही `2025-11-25` साठी वैध). [धड्यासाठी येथे क्लिक करा](./14-sampling/README.md)

- **15 MCP ऍप्स**. असे MCP सर्व्हर तयार करा जे UI सूचना देखील पाठवतात, [धड्यासाठी येथे क्लिक करा](./15-mcp-apps/README.md)

मॉडेल कॉन्टेक्स्ट प्रोटोकॉल (MCP) हा एक उघड प्रोटोकॉल आहे जो अनुप्रयोगांना LLMs ला संदर्भ कसा द्यायचा याचे मानकीकरण करतो. MCP ला AI अनुप्रयोगांसाठी USB-C पोर्ट समजा - तो AI मॉडेल्सना विविध डेटा स्रोत आणि साधनांशी जोडण्याचा मानकीकृत मार्ग प्रदान करतो.

## शिकण्याचे उद्दिष्टे

या धड्याच्या शेवटी, तुम्ही खालील गोष्टी करू शकाल:

- MCP साठी C#, Java, Python, TypeScript, आणि JavaScript मध्ये विकास पर्यावरण स्थापन करा
- कस्टम वैशिष्ट्यांसह (संसाधने, प्रॉम्प्ट, आणि साधने) मूलभूत MCP सर्व्हर तयार करा आणि तैनात करा
- होस्ट अनुप्रयोग तयार करा जे MCP सर्व्हरसह कनेक्ट होतात
- MCP अंमलबजावणी तपासणी आणि डीबग करा
- सामान्य सेटअप आव्हाने आणि त्यांची सोडवणूक समजून घ्या
- तुमच्या MCP अंमलबजावणींना लोकप्रिय LLM सेवांशी कनेक्ट करा

## तुमचे MCP पर्यावरण सेट करणे

MCP सह काम करण्यापूर्वी, तुमचे विकास वातावरण तयार करणे आणि मूलभूत कार्यप्रवाह समजून घेणे महत्त्वाचे आहे. या विभागात सुरुवातीचे सेटअप पावले तुम्हाला मार्गदर्शन करतील जेणेकरून तुमच्या MCP सह सुरळीत सुरुवात होईल.

### पूर्वपरिस्थिती

MCP विकासात उतरण्यापूर्वी, हे सुनिश्चित करा की तुमच्याकडे आहे:

- **विकास पर्यावरण**: तुमच्या निवडलेल्या भाषेसाठी (C#, Java, Python, TypeScript, किंवा JavaScript)
- **IDE/एडिटर**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm, किंवा कोणताही आधुनिक कोड एडिटर
- **पॅकेज मॅनेजर्स**: NuGet, Maven/Gradle, pip, किंवा npm/yarn
- **API की**: कोणत्याही AI सेवा वापरण्यासाठी जे तुम्ही होस्ट अनुप्रयोगांमध्ये वापरणार आहात


### अधिकृत SDKs

पुढील प्रकरणांमध्ये तुम्हाला Python, TypeScript, Java आणि .NET वापरून तयार केलेली उपयोजन दिसतील. येथे सर्व अधिकृतपणे समर्थित SDKs दिले आहेत.

MCP अनेक भाषांसाठी अधिकृत SDKs प्रदान करतो ([MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/) प्रमाणे):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - Microsoft सह सहकार्याने देखभाल करण्यात येते
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI सह सहकार्याने देखभाल
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - अधिकृत TypeScript अंमलबजावणी
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - अधिकृत Python अंमलबजावणी (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - अधिकृत Kotlin अंमलबजावणी
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI सह सहकार्याने देखभाल
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - अधिकृत Rust अंमलबजावणी
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - अधिकृत Go अंमलबजावणी

## मुख्य मुद्दे

- भाषा-विशिष्ट SDKs वापरून MCP विकास वातावरण सेट करणे सोपे आहे
- MCP सर्व्हर तयार करणे म्हणजे स्पष्ट स्कीमा असलेल्या साधनांची निर्मिती आणि नोंदणी करणे
- MCP क्लायंट्स सर्व्हर आणि मॉडेल्सशी कनेक्ट होतात जेणेकरून विस्तारित क्षमता वापरता येतात
- तपासणी आणि डीबगिंग विश्वसनीय MCP अंमलबजावणीसाठी आवश्यक आहे
- तैनातीच्या पर्यायांमध्ये स्थानिक विकासापासून ते क्लाऊड-आधारित सोल्यूशन्सपर्यंत विविधता आहे
