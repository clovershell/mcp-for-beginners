# ប្រធានបទកម្រិតខ្ពស់នៅក្នុង MCP

[![Advanced MCP: Secure, Scalable, and Multi-modal AI Agents](../../../translated_images/km/06.42259eaf91fccfc6.webp)](https://youtu.be/4yjmGvJzYdY)

_(ចុចរូបភាពខាងលើដើម្បីមើលវីដេអូអំពីមេរៀននេះ)_

មេរៀននេះគ្របដណ្តប់ប្រធានបទកម្រិតខ្ពស់ជាច្រើនក្នុងការអនុវត្ត Model Context Protocol (MCP) រួមមាន ការរួមបញ្ចូលបច្ចេកទេសពហុម៉ូដល, ការកែលម្អភាពសមត្ថភាពខ្ពស់, វិធានការ​សុវត្ថិភាព​ល្អបំផុត និងការរួមបញ្ចូលរបស់សហគ្រាស។ ប្រធានបទទាំងនេះមានសារៈសំខាន់សម្រាប់បង្កើតកម្មវិធី MCP ដែលមានស្ថេរភាព និងឆាប់រហ័សក្នុងការផលិត ដែលអាចផ្តល់ជូនតម្រូវការរបស់ប្រព័ន្ធ AI សម័យទំនើប។

## ទិដ្ឋភាពទូទៅ

មេរៀននេះស្វែងយល់ពីគំនិតកម្រិតខ្ពស់នៅក្នុងការអនុវត្ត Model Context Protocol ជាមួយការផ្ដោតលើការរួមបញ្ចូលជាមួយម៉ូដលពហុ, ភាពអាចបត់បែនបានខ្ពស់, របៀបអនុវត្តវិធានការ​សុវត្ថិភាពល្អបំផុត និងការរួមបញ្ចូលក្នុងបរិបទសហគ្រាស។ ប្រធានបទទាំងនេះគឺពេញនិយមសម្រាប់បង្កើតកម្មវិធី MCP ដែលមានគុណភាពផលិតកម្ម ដែលអាចដោះស្រាយតម្រូវការលំបាកនៅក្នុងបរិស្ថានសហគ្រាស។

> **មើលទៅមុខ៖** ប្រធានបទជាច្រើនខាងក្រោម ដូចជាផ្នែក Root Contexts (5.4) និង Sampling (5.6) ត្រូវបានវាយតម្លៃដោយការចេញផ្សាយជាកម្មវិធីជាការប្រកាស MCP នៅថ្ងៃទី `2026-07-28` ដែលបានដាក់ទៅលើគោលការណ៍ដែលត្រូវបានកំនត់ថាផុតកំណត់ ហើយមុខងារ Tasks ដែលត្រូវបានសាកល្បង ហៅបញ្ចូលក្នុង Protocol Features (5.16) ត្រូវបានផ្លាស់ទីទៅនូវផ្នែក Tasks ផ្ទាល់ខ្លួន។ សូមមើល [មានអ្វីដែលកំពុងផ្លាស់ប្តូរនៅ MCP៖ ការប្រកាសជាការប្រកាស `2026-07-28`](../01-CoreConcepts/mcp-2026-07-28-release-candidate.md) សម្រាប់ព័ត៌មានលម្អិត។

## គោលបំណងការសិក្សា

នៅចុងបញ្ចប់មេរៀននេះ អ្នកនឹងអាច:

- រួមបញ្ចូលមុខងារពហុម៉ូដលក្នុងសហគ្រាស MCP
- រចនាស្ថាបត្យកម្ម MCP អាចបត់បែនបានសម្រាប់ស្ថានភាពតម្រូវខ្ពស់
- អនុវត្តវិធានការ​សុវត្ថិភាពល្អបំផុត ដែលសមស្របជាមួយគោលការណ៍សុវត្ថិភាពរបស់ MCP
- រួមបញ្ចូល MCP ជាមួយប្រព័ន្ធ AI និងស៊េរីសហគ្រាស
- បង្កើនសមត្ថភាព និងភាពជឿជាក់នៅក្នុងបរិយាកាសផលិតកម្ម

> **ថ្មីក្នុងកម្រិតឯកសារពិស្ដារ MCP 2025-11-25**៖ ឯកសារពិស្ដារបច្ចុប្បន្នបានរួមបញ្ចូលការគាំទ្រជាសាកល្បងសម្រាប់ **កិច្ចការនានា**(ដំណើរការជារយៈពេលវែងមានការតាមដានជំហាន), **ការរក្សាអនុស្សាវរីយ៍ឧបករណ៍**(metadata អំពីវិធានការអនុវត្តឧបករណ៍សម្រាប់សុវត្ថិភាព), **ការទទួលយកមាតិកាURL**(ការស្នើសុំមាតិកាURL ពិសេសពីភាគីអតិថិជន), និង **Root** (សម្រាប់ការគ្រប់គ្រង context នៃកន្លែងការងារ) បានកើនឡើង។ សូមមើល [បញ្ជីការផ្លាស់ប្ដូរក្នុងឯកសារពិស្ដារម៉ូដែល context protocol](https://spec.modelcontextprotocol.io/) សម្រាប់ព័ត៌មានលម្អិត។

## ផែនការហាត់ប្រាណ

រចនាកម្មវិធី MCP កម្រិតសហគ្រាសសម្រាប់ករណីប្រើប្រាស់ជាក់លាក់មួយ៖

2. រៀបចំវិធានការសុវត្ថិភាពដែលចាំបាច់ដើម្បីការពារទិន្នន័យដើម្បីភាពឯកជន
3. រចនាស្ថាបត្យកម្មដែលអាចបត់បែនសម្រាប់ទម្ងន់បំផ្លាញនានា
4. ប្លង់ចំណុចរួមបញ្ចូលជាមួយប្រព័ន្ធ AI សហគ្រាស
5. ចុះកំណត់បួនករណីបញ្ហាសមត្ថភាព និងយុទ្ធសាស្រ្តបំបាត់វា
