## ការចាប់ផ្តើម  

[![Build Your First MCP Server](../../../translated_images/km/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(ចុចលើរូបភាពខាងលើដើម្បីមើលវីដេអូមេរៀននេះ)_

ផ្នែកនេះមានមេរៀនជាច្រើន៖

- **1 ម៉ាស៊ីនមិត្តភក្តិដំបូងរបស់អ្នក** នៅក្នុងមេរៀនដំបូងនេះ អ្នកនឹងរៀនពីវិធីបង្កើតម៉ាស៊ីនមិត្តភក្តិដំបូងរបស់អ្នក និងពិនិត្យវាតាមឧបករណ៍ inspector ដែលជាវិធីមានតម្លៃក្នុងការសាកល្បង និងដោះសោភាពសារៈសំខាន់របស់ម៉ាស៊ីនមិត្តភក្តិរបស់អ្នក, [ទៅមេរៀន](01-first-server/README.md)

- **2 ក្លាយជាអតិថិជន**, ក្នុងមេរៀននេះ អ្នកនឹងរៀនពីវិធីសរសេរអតិថិជនដែលអាចភ្ជាប់ទៅម៉ាស៊ីនមិត្តភក្តិរបស់អ្នក, [ទៅមេរៀន](02-client/README.md)

- **3 អតិថិជនជាមួយ LLM**, វិធីល្អខ្ពស់ជាងនៃការសរសេរអតិថិជនគឺដោយបន្ថែម LLM ទៅក្នុងវា ដូច្នេះវាអាច "ចរចារ" ជាមួយម៉ាស៊ីនមិត្តភក្តិរបស់អ្នកថាតើត្រូវធ្វើអ្វី, [ទៅមេរៀន](03-llm-client/README.md)

- **4 ការប្រើមុខងារ GitHub Copilot Agent របស់ម៉ាស៊ីនមិត្តភក្តិក្នុង Visual Studio Code**។ នៅទីនេះ យើងកំពុងមើលពីការប្រតិបត្តិម៉ាស៊ីនមិត្តភក្តិ MCP ពីក្នុង Visual Studio Code, [ទៅមេរៀន](04-vscode/README.md)

- **5 ម៉ាស៊ីនមិត្តភក្តិ stdio Transport** stdio transport គឺជាម៉ាស៊ីននិយមសម្រាប់ការទំនាក់ទំនងរវាងម៉ាស៊ីនមិត្តភក្តិ MCP និងអតិថិជននៅក្នុងតំបន់តែមួយ ផ្ដល់ការទំនាក់ទំនង subprocess ដែលមានសុវត្ថិភាពជាមួយការបែកញែកដំណើរការដែលស្រាប់ [ទៅមេរៀន](05-stdio-server/README.md)

- **6 ការផ្ដាច់ផ្សាយ HTTP ជាមួយ MCP (Streamable HTTP)**។ រៀនអំពីការផ្ដាច់ផ្សាយប្រព័ន្ធ HTTP ម៉ូដថ្មី (វិធីសាស្ត្រដែលបានណែនាំសម្រាប់ម៉ាស៊ីនមិត្តភក្តិ MCP ពីចម្ងាយតាម [MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http)), ការជូនដំណឹងអំពីដំណើរការនិងវិធីសាស្ត្រទំនើបក្នុងការអនុវត្តម៉ាស៊ីនមិត្តភក្តិ និង អតិថិជន MCP ពីរោងបរិក្ខារផ្ដាច់ផ្សាយ HTTP. [ទៅមេរៀន](06-http-streaming/README.md)

- **7 ការប្រើប្រាស់ AI Toolkit សម្រាប់ VSCode** ដើម្បីប្រើ និងសាកល្បងអតិថិជន និងម៉ាស៊ីនមិត្តភក្តិ MCP របស់អ្នក [ទៅមេរៀន](07-aitk/README.md)

- **8 ការសាកល្បង**។ នៅលើនេះ យើងនឹងផ្តោតនៅលើវិធីសាស្ត្រផ្សេងៗក្នុងការសាកល្បងម៉ាស៊ីនមិត្តភក្តិ និងអតិថិជនរបស់យើង, [ទៅមេរៀន](08-testing/README.md)

- **9 ការចេញផ្សាយ**។ ជាប់ចង្វាក់នេះនឹងមើលពីវិធីផ្សេងៗនៃការចេញផ្សាយដំណោះស្រាយ MCP របស់អ្នក, [ទៅមេរៀន](09-deployment/README.md)

- **10 ការប្រើប្រាស់ម៉ាស៊ីនមិត្តភក្តិលំដាប់ខ្ពស់**។ ជាប់ចង្វាក់នេះពិភាក្សាអំពីការប្រើប្រាស់ខ្ពស់របស់ម៉ាស៊ីនមិត្តភក្តិ, [ទៅមេរៀន](./10-advanced/README.md)

- **11 ការផ្តល់សិទ្ធិ**។ ជាប់ចង្វាក់នេះនឹងពិភាក្សាអំពីរបៀបបន្ថែមការផ្តល់សិទ្ធិសាមញ្ញចាប់ពី Basic Auth ដល់ការប្រើ JWT និង RBAC។ អ្នកត្រូវបានលើកទឹកចិត្តឲ្យចាប់ផ្តើមនៅទីនេះ ហើយបន្ទាប់មកមើលប្រធានបទលំដាប់ខ្ពស់នៅជំពូក 5 និងអនុវត្តការពារជាថ្មីតាមការផ្តល់អនុសាសន៍នៅជំពូក 2, [ទៅមេរៀន](./11-simple-auth/README.md)

- **12 ម៉ាស៊ីនមិត្តភក្តិ MCP**។ កំណត់តម្លៃ និងប្រើប្រាស់អតិថិជនម៉ាស៊ីនមិត្តភក្តិ MCP ដ៏ពេញនិយមរួមមាន Claude Desktop, Cursor, Cline និង Windsurf។ រៀនអំពីប្រភេទការដឹកជញ្ជូន និងការដោះស្រាយបញ្ហា, [ទៅមេរៀន](./12-mcp-hosts/README.md)

- **13 MCP Inspector**។ ដោះសោភាពសម្រាប់ម៉ាស៊ីនមិត្តភក្តិ MCP របស់អ្នកយ៉ាងអន្តរជាតិដោយប្រើឧបករណ៍ MCP Inspector។ រៀនដោះស្រាយឧបករណ៍ ប្រភពធនធាន និងសារប្រភេទប្រព័ន្ធ, [ទៅមេរៀន](./13-mcp-inspector/README.md)

- **14 មាតិកាគំរូ**។ បង្កើតម៉ាស៊ីនមិត្តភក្តិ MCP ដែលសហការជាមួយអតិថិជន MCP លើភារកិច្ចដែលពាក់ព័ន្ធនឹង LLM (ត្រូវបានលុបចោលនៅក្នុងជម្រើសចេញផ្សាយ `2026-07-28`; នៅតែមានប្រសិទ្ធភាពសម្រាប់ `2025-11-25`). [ទៅមេរៀន](./14-sampling/README.md)

- **15 កម្មវិធី MCP**។ សាងសង់ម៉ាស៊ីនមិត្តភក្តិ MCP ដែលឆ្លើយតបជាមួយសេចក្តីណែនាំ UI ផងដែរ, [ទៅមេរៀន](./15-mcp-apps/README.md)

ពិធីការម៉ូដែល Context Protocol (MCP) គឺជាពិធីការលំហដែលលំដាប់ការបញ្ចូលcontext ទៅឲ្យ LLMs។ គិតថា MCP គឺដូចជា​ព្រលឹង USB-C សម្រាប់កម្មវិធី AI - វាផ្ដល់វិធីសាស្ត្រមានលំនាំដើម្បីភ្ជាប់ម៉ូដែល AI ទៅប្រភពទិន្នន័យ និងឧបករណ៍ផ្សេងៗ។

## គោលបំណងសិក្សា

នៅចុងបញ្ចប់មេរៀននេះ អ្នកនឹងអាច៖

- កំណត់បរិស្ថានអភិវឌ្ឍសម្រាប់ MCP ក្នុងភាសា C#, Java, Python, TypeScript, និង JavaScript
- បង្កើត និងចេញផ្សាយម៉ាស៊ីនមិត្តភក្តិ MCP តុល្យភាពមូលដ្ឋានជាមួយលក្ខណៈពិសេសផ្ទាល់ខ្លួន (ធនធាន, ពិន្ទុប្រមូល, និងឧបករណ៍)
- បង្កើតកម្មវិធីម៉ាស៊ីនមិត្តភក្តិដែលភ្ជាប់ទៅម៉ាស៊ីនមិត្តភក្តិ MCP
- សាកល្បង និងដោះសោភាពការអនុវត្ត MCP
- យល់ដឹងពីបញ្ហាការកំណត់ និងដំណោះស្រាយធម្មតា
- ភ្ជាប់អនុវត្ត MCP របស់អ្នកទៅកាន់សេវាកម្ម LLM ដែលពេញនិយម

## កំណត់បរិស្ថាន MCP របស់អ្នក

មុននឹងចាប់ផ្តើមធ្វើការជាមួយ MCP វាសំខាន់ក្នុងការរៀបចំបរិស្ថានអភិវឌ្ឍ និងយល់ពីដំណើរការស្នូល។ ផ្នែកនេះនឹងណែនាំអ្នកក្នុងជំហានរៀបចំដំបូងដើម្បីធានាការចាប់ផ្តើមអោយរលូនជាមួយ MCP។

### ខ្លឹមសារជាមុន

មុនយោងមកការអភិវឌ្ឍ MCP សូមប្រាកដថាអ្នកមាន៖

- **បរិស្ថានអភិវឌ្ឍ**៖ សម្រាប់ភាសាដែលអ្នកបានជ្រើសរើស (C#, Java, Python, TypeScript, ឬ JavaScript)
- **IDE/កម្មវិធីកែសម្រួល**៖ Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm ឬកម្មវិធីកែសម្រួលកូដទំនើបណាមួយ
- **អ្នកគ្រប់គ្រងកញ្ចប់**៖ NuGet, Maven/Gradle, pip, ឬ npm/yarn
- **កូនសោ API**៖ សម្រាប់សេវាកម្ម AI ដែលអ្នកគ្រោងចង់ប្រើនៅក្នុងកម្មវិធីម៉ាស៊ីនមិត្តភក្តិរបស់អ្នក


### SDKs ផ្លូវការនានា

នៅជំពូកក្រោយ អ្នកនឹងឃើញដំណោះស្រាយដែលបានសាងសង់ដោយប្រើ Python, TypeScript, Java និង .NET។ ខាងក្រោមជារាយនាម SDKs ផ្លូវការដែលគាំទ្រ។

MCP ផ្ដល់ SDKs ផ្លូវការសម្រាប់ភាសាច្រើន (ស្របតាម [MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/)):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - រក្សាបានដោយសហការជាមួយ Microsoft
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - រក្សាបានដោយសហការជាមួយ Spring AI
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - ការអនុវត្តផ្លូវការរបស់ TypeScript
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - ការអនុវត្តផ្លូវការរបស់ Python (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - ការអនុវត្តផ្លូវការរបស់ Kotlin
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - រក្សាបានដោយសហការជាមួយ Loopwork AI
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - ការអនុវត្តផ្លូវការរបស់ Rust
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - ការអនុវត្តផ្លូវការរបស់ Go

## ចំណុចសំខាន់

- ការរៀបចំបរិស្ថានអភិវឌ្ឍ MCP គឺស្រួលជាមួយ SDKs ដែលមានលក្ខណៈផ្តល់ជូនសម្រាប់ភាសារៀងខ្លួន
- ការសាងសង់ម៉ាស៊ីនមិត្តភក្តិ MCP រួមបញ្ចូលការបង្កើត និងចុះបញ្ជីឧបករណ៍ជាមួយស្ពង់ច្បាស់
- អតិថិជន MCP ភ្ជាប់ទៅម៉ាស៊ីនមិត្តភក្តិ និងម៉ូដែលដើម្បីប្រើប្រាស់សមត្ថភាពបន្ថែម
- ការសាកល្បង និងដោះសោភាពគឺជាកត្តាសំខាន់សម្រាប់ការអនុវត្ត MCP ដែលទុកចិត្តបាន
- ជម្រើសចេញផ្សាយមានចាប់ពីការអភិវឌ្ឍនៅក្នុងតំបន់ដល់ដំណោះស្រាយលើពពក
