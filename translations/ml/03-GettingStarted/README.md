## ആരംഭിക്കുന്നത്  

[![Build Your First MCP Server](../../../translated_images/ml/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(ഈ പാഠത്തിന്റെ വീഡിയൊ കാണാൻ മുകളിൽ ചിത്രത്തിൽ ക്ലിക്ക് ചെയ്യു)_

ഈ വിഭാഗത്തിൽ പല പാഠങ്ങളും ഉൾപ്പെടുന്നു:

- **1 നിങ്ങളുടെ ആദ്യ സെർവർ**, ഈ ആദ്യ പാഠത്തിൽ, നിങ്ങളുടെ ആദ്യ സെർവർ എങ്ങിനെ സൃഷ്ടിച്ച് ഇംസ്പെക്ടർ ടൂൾ ഉപയോഗിച്ച് പരിശോധിക്കാം എന്ന് പഠിക്കും, സെർവർ ടെസ്റ്റ് ചെയ്യാനും ഡീബഗ് ചെയ്യാനുമുള്ള ഒരു മൂല്യവാന മാർഗം, [പാഠത്തിലേക്ക്](01-first-server/README.md)

- **2 ക്ലയന്റ്**, ഈ പാഠത്തിൽ, നിങ്ങളുടെ സെർവറിൽ കണക്റ്റ് ചെയ്യാൻ കഴിയുന്ന ഒരു ക്ലയന്റ് എങ്ങിനെ എഴുതാം എന്നത് പഠിക്കും, [പാഠത്തിലേക്ക്](02-client/README.md)

- **3 LLMഉണ്ട് ക്ലയന്റ്**, മറ്റൊരു ശ്രേഷ്ഠ മാർഗം LLM ചേർത്ത് ഒരു ക്ലയന്റ് എഴുതുന്നതിലൂടെ അതു സെർവറുമായി "നേഗോഷ്യേഷൻ" നടത്തുന്നു എന്നതാണ്, [പാഠത്തിലേക്ക്](03-llm-client/README.md)

- **4 Visual Studio Code-യിൽ GitHub Copilot ഏജന്റ് മോഡ് ഉപയോഗിച്ച് MCP സെർവർ ഉപയോഗിക്കൽ**. ഇവിടെ, Visual Studio Code-യിൽനിന്ന് MCP സെർവർ ഓടിക്കൽ ചർച്ച ചെയ്യുന്നു, [പാഠത്തിലേക്ക്](04-vscode/README.md)

- **5 stdio ട്രാൻസ്പോർട്ട് സെർവർ** stdio ട്രാൻസ്പോർട്ട് ലൊക്കൽ MCP സെർവർ-ക്ലയന്റ് കമ്മ്യൂണിക്കേഷനായി ശുപാർശ ചെയ്ത സ്റ്റാൻഡേർഡ് ആണ്, ഇത് ഉൾപ്പെട്ട പ്രോസസ് ഐസൊലേഷനോടു കൂടിയ സുരക്ഷിത subprocess അടിസ്ഥാനത്തിലുള്ള കമ്മ്യൂണിക്കേഷൻ നൽകുന്നു, [പാഠത്തിലേക്ക്](05-stdio-server/README.md)

- **6 MCP ഉപയോഗിച്ചുള്ള HTTP സ്റ്റ്രീമിംഗ് (Streamable HTTP)**. ആധുനിക HTTP സ്റ്റ്രീമിംഗ് ട്രാൻസ്പോർട്ട് (2025-11-25 MCP നിർദ്ദേശപ്രകാരം റിമോട്ട് MCP സെർവർസ്‌ക്കായി ശുപാർശ ചെയ്ത മാർഗം), പുരോഗതി അറിയിപ്പുകൾ, സ്‌കെയിലബിളും റിയൽ-ടൈം MCP സെർവർ ക്ലയന്റ് നിർമാണം Streamable HTTP ഉപയോഗിച്ച് പഠിക്കുക, [പാഠത്തിലേക്ക്](06-http-streaming/README.md)

- **7 VSCode-ൽ AI ടൂൾകിറ്റ് ഉപയോഗിക്കൽ** MCP ക്ലയന്റുകളും സെർവറുകളും പരീക്ഷിക്കുകയും ഉപയോഗിക്കുകയും ചെയ്യുന്നതിന് [പാഠത്തിലേക്ക്](07-aitk/README.md)

- **8 പരിശോധന**. ഇവിടെ, വിവിധ മാർഗങ്ങളിൽ സെർവർ, ക്ലയന്റ് ടെസ്റ്റിംഗ് സവിശേഷമായി ശ്രദ്ധിക്കുന്നു, [പാഠത്തിലേക്ക്](08-testing/README.md)

- **9 വിന്യാസം**. MCP പരിഹാരങ്ങൾ വിന്യസിക്കുന്ന വ്യത്യസ്ത മാർഗങ്ങൾ പരിശോധിക്കും, [പാഠത്തിലേക്ക്](09-deployment/README.md)

- **10 ഉയർന്ന തലത്തിലുള്ള സെർവർ ഉപയോഗം**. ഈ അദ്ധ്യായം ഉയർന്ന തലത്തിലുള്ള സെർവർ ഉപയോഗം ഉൾക്കൊള്ളുന്നു, [പാഠത്തിലേക്ക്](./10-advanced/README.md)

- **11 അംഗത്വം**. സിമ്പിൾ അംഗത്വം ചേർക്കുന്ന മാർഗങ്ങൾ ഉൾപ്പെടുന്നു, ബേസിക് അംഗത്വം മുതൽ JWT, RBAC വരെ. ആദ്യം ഇവിടെ ആരംഭിച്ച് പിന്നീട് അദ്ധ്യായം 5 ലെ ഉയർന്ന തല വിഷയങ്ങൾ കാഴ്‌ചവെച്ച്, അയ്ഞ്ച്യായം 2 ലെ ശുപാർശകളിലൂടെ പിന്തുടർന്ന് സുരക്ഷ കൂടുതൽ ശക്തമാക്കാൻ ഉദ്ദേശിക്കുന്നു, [പാഠത്തിലേക്ക്](./11-simple-auth/README.md)

- **12 MCP ഹോസ്റ്റുകൾ**. Claude Desktop, Cursor, Cline, Windsurf എന്നിവ ഉൾപ്പെടെയുള്ള പ്രശസ്ത MCP ഹോസ്റ്റ് ക്ലയന്റുകൾ ക്രമീകരിക്കുകയും ഉപയോഗിക്കുകയും ചെയ്യുക. ട്രാൻസ്പോർട്ട് തരം, പ്രശ്‌നപരിഹാരം എന്നിവ പഠിക്കുക, [പാഠത്തിലേക്ക്](./12-mcp-hosts/README.md)

- **13 MCP ഇൻസ്പെക്ടർ**. MCP ഇൻസ്പെക്ടർ ടൂൾ ഉപയോഗിച്ച് ഇന്ററാക്റ്റീവ് ആകെയുള്ള MCP സെർവർ ഡീബഗും ടെസ്റ്റിംഗും. ടൂൾസ്, സ്രോതസ്സുകൾ, പ്രോട്ടോക്കോൾ സന്ദേശങ്ങൾ പ്രശ്‌നപരിഹാര ഉപാധികൾ പഠിക്കുക, [പാഠത്തിലേക്ക്](./13-mcp-inspector/README.md)

- **14 സാമ്പ്ലിംഗ്**. LLM ബന്ധപ്പെട്ട പ്രവർത്തനങ്ങളിൽ MCP ക്ലയന്റുകളുമായി സഹകരിക്കുന്ന MCP സെർവർസ് സൃഷ്ടിക്കുക (2026-07-28 റിലീസ് സ്ഥാനാർത്ഥിയിൽ അസാധുവായി; 2025-11-25തിന് പ്രസക്തം), [പാഠത്തിലേക്ക്](./14-sampling/README.md)

- **15 MCP ആപ്ലിക്കേഷനുകൾ**. UI നിർദ്ദേശങ്ങളോടൊപ്പം മറുപടി നൽകുന്ന MCP സെർവർ നിർമ്മിക്കുക, [പാഠത്തിലേക്ക്](./15-mcp-apps/README.md)

മോഡൽ കോൺടക്സ്റ്റ് പ്രോട്ടോക്കോൾ (MCP) ഒരു തുറന്ന പ്രോട്ടോക്കോൾ ആണ്, ആപ്ലിക്കേഷനുകൾ എങ്ങനെ LLM-കൾക്ക് കോൺടക്സ്റ്റ് നൽകും എന്നത് സ്റ്റാൻഡേർഡ് ചെയ്യുന്ന. MCP AI ആപ്ലിക്കേഷനുകൾക്കുള്ള USB-C പോർട്ട് പോലെയാണ് - AI മോഡലുകൾ വ്യത്യസ്ത ഡേറ്റാ സ്രോതസ്സുകളുമായി ടൂളുകളുമായി എങ്ങനെ കണക്റ്റ് ചെയ്യാമെന്നതു ഒരു സ്റ്റാൻഡേർഡ് മാർഗം നൽകുന്നു.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ പാഠം അവസാനിച്ചിരുന്നു വരെ, നിങ്ങൾക്ക് കഴിയും:

- C#, ജാവ, പൈതൺ, ടൈ‌പ്സ്ക്രിപ്റ്റ്, ജാവാസ്ക്രിപ്റ്റ് എന്നിവയിൽ MCP വികസന പരിസരങ്ങൾ സജ്ജമാക്കുക
- കസ്റ്റം സവിശേഷതകളോടെ (റിസോഴ്‌സുകൾ, പ്രോംപ്റ്റുകൾ, ടൂളുകൾ) അടിസ്ഥാന MCP സെർവറുകൾ നിർമ്മിക്കയും വിന്യസിക്കയും ചെയ്യുക
- MCP സെർവറുകളുമായി ബന്ധിപ്പിക്കുന്ന ഹോസ്റ്റ് ആപ്ലിക്കേഷനുകൾ സൃഷ്ടിക്കുക
- MCP നടപ്പാക്കലുകൾ ടെസ്റ്റ് ചെയ്യുകയും ഡീബഗ് ചെയ്യുകയും ചെയ്യുക
- സാധാരണ സജ്ജീകരണ വെല്ലുവിളികളും അവയുടെ പരിഹാരങ്ങളും മനസിലാക്കുക
- പ്രശസ്ത LLM സേവനങ്ങളുമായി MCP നടപ്പാക്കലുകൾ കണക്റ്റ് ചെയ്യുക

## നിങ്ങളുടെ MCP പരിസരം സജ്ജമാക്കൽ

MCP ഉപയോഗിച്ച് പ്രവർത്തനം ആരംഭിക്കുന്നതിന് മുമ്പായി, വികസന പരിസരം തയ്യാറാക്കുകയും അടിസ്ഥാന പ്രവർത്തി പ്രവാഹം മനസ്സിലാക്കുകയും ചെയ്യേണ്ടത് പ്രധാനമാണ്. ഈ സെക്ഷൻ MCP-യുമായി സുലഭമായ ആരംഭം ഉറപ്പാക്കാൻ ആരംഭവർക്കും മാർഗ്ഗനിർദ്ദേശം നൽകും.

### മുൻകൂട്ടി അംഗീകാരം

MCP വികസനത്തിലേക്ക് കടക്കുന്നതിന് മുമ്പായി, ഉറപ്പാക്കുക നിങ്ങൾക്കുണ്ട്:

- **വികസന പരിസരം**: നിങ്ങളുടെ തിരഞ്ഞെടുത്ത ഭാഷ (C#, Java, Python, TypeScript, JavaScript)
- **IDE/എഡിറ്റർ**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm അല്ലെങ്കിൽ ഏതെങ്കിലും ആധുനിക കോഡ് എഡിറ്റർ
- **പാക്കേജ് മാനേജറുകൾ**: NuGet, Maven/Gradle, pip, അല്ലെങ്കിൽ npm/yarn
- **API കീകൾ**: ഹോസ്റ്റ് ആപ്ലിക്കേഷനുകളിൽ ഉപയോഗിക്കുക എന്ന് പദ്ധതിയിടുന്ന ഏതെങ്കിലും AI സേവനങ്ങൾക്ക്


### ഔദ്യോഗിക SDKകൾ

വരാനിരിക്കുന്ന അദ്ധ്യായങ്ങളിൽ Python, TypeScript, Java, .NET ഉപയോഗിച്ച് നിർമ്മിച്ച പരിഹാരങ്ങൾ കാണാൻ കഴിയും. ഇവിടെ എല്ലാ ഔദ്യോഗികമായി പിന്തുണയ്ക്കുന്ന SDK കളും ചേർക്കുന്നു.

MCP പല ഭാഷകൾക്കായി ഔദ്യോഗിക SDKകൾ നൽകുന്നു ([MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/) പ്രകാരം):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - Microsoft-ന്റെ സഹകരണത്തോടെയാണ് പരിപാലനം
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI-യുമായി സഹകരിച്ച് പരിപാലിക്കുന്നു
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - ഔദ്യോഗിക TypeScript നടപ്പാക്കൽ
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - ഔദ്യോഗിക Python നടപ്പാക്കൽ (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - ഔദ്യോഗിക Kotlin നടപ്പാക്കൽ
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI-യുടെ സഹകരണത്തോടെ പരിപാലണം
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - ഔദ്യോഗിക Rust നടപ്പാക്കൽ
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - ഔദ്യോഗിക Go നടപ്പാക്കൽ

## പ്രധാന പഠിപ്പുകൾ

- ഭാഷാസമ്പ്രദായം SDK ഗൾ ഉപയോഗിച്ച് MCP വികസന പരിസരം സജ്ജമാക്കുക എളുപ്പമാണ്
- MCP സെർവർ നിർമ്മിക്കുന്നതിൽ ലളിതമായ സ്കീമകളോടെ ടൂളുകൾ സൃഷ്ടിക്കുകയും രജിസ്റ്ററുചെയ്യുകയും ചെയ്യണം
- MCP ക്ലയന്റുകൾ സെർവർ കളുടെയും മോഡലുകളുടെയും വിശേഷതകൾ പ്രയോജനപ്പെടുത്താൻ ബന്ധിപ്പിക്കുന്നു
- വിശ്വസനീയമായ MCP നടപ്പാക്കലിന് പരിശോധനയും ഡീബഗും അനിവാര്യമാണ്
- വിന്യാസം ലൊക്കൽ ഡെവലപ്മെന്റുവിൽ നിന്നും ക്ലൗഡ് അടിസ്ഥാനപരമായ പരിഹാരങ്ങളിലേക്കും വ്യാപിച്ചു
