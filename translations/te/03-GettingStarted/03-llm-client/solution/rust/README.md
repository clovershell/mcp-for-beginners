# ఈ నమూనా నడపడం

ఇది LLM క్లయింట్ సాంపిల్ కోసం Rust పరిష్కారం. మీరు Rust టూల్‌చైన్ ఇన్స్టాల్ చేయాలి; [అధికారిక ఇన్స్టాల్ గైడ్](https://www.rust-lang.org/tools/install) చూడండి.

క్లయింట్ GitHub మోడల్స్ అనుమాన ఎండ్పాయింట్ (`https://models.github.ai/inference/chat`) ద్వారా ఒక మోడల్‌ను పిలుస్తుంది మరియు మీ GitHub వ్యక్తిగత యాక్సెస్ టోకెన్ (PAT)ను `OPENAI_API_KEY` పరిసర వేరియబుల్ నుండి చదువుతుంది.

> [!NOTE]
> ఈ రిపోలోని ఇతర పరిష్కారాలు `GITHUB_TOKEN` ఉపయోగిస్తాయి. Rust కోసం, OpenAI క్లయింట్ కాన్ఫిగరేషన్ ని సరిపోల్చేందుకు `OPENAI_API_KEY` ని అదే విలువకు సెట్ చేయండి.

## -0- మీ GitHub టోకెన్ సెట్ చేయండి

```bash
# జెడ్‌ఎస్‌హెచ్/బాష్
export OPENAI_API_KEY="{{YOUR_GITHUB_PAT}}"
```

```powershell
# పవర్‌షెల్
$env:OPENAI_API_KEY = "{{YOUR_GITHUB_PAT}}"
```

## -1- నమూనాను నిర్మించండి

```bash
cargo build
```

## -2- నమూనాను నడిపించండి

```bash
cargo run
```

క్లయింట్ క్యాల్క్యులేటర్ MCP సర్వర్ ప్రారంభించుతుంది, దాని టూల్ జాబితాను తీసుకువస్తుంది, మరియు `add` టూల్ పిలవడానికి మోడల్ (`openai/gpt-5-mini`) ఉపయోగిస్తుంది. మీరు టూల్ పిలుపు (ఉదాహరణకు, "Calling tool: add") మరియు ఆ పిలుపు ఫలితాన్ని చూపుతుందని చూడాలి.
