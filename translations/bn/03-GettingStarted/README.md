## শুরু করা  

[![আপনার প্রথম MCP সার্ভার তৈরি করুন](../../../translated_images/bn/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(এই পাঠের ভিডিওটি দেখতে উপরের ছবিতে ক্লিক করুন)_

এই অংশে কয়েকটি পাঠ অন্তর্ভুক্ত:

- **1 আপনার প্রথম সার্ভার**, প্রথম এই পাঠে আপনি শিখবেন কীভাবে আপনার প্রথম সার্ভার তৈরি করতে হয় এবং ইনস্পেক্টর টুল দিয়ে এটি পরিদর্শন করতে হয়, যা আপনার সার্ভার পরীক্ষণ এবং ডিবাগের জন্য একটি মূল্যবান উপায়, [পাঠটি দেখুন](01-first-server/README.md)

- **2 ক্লায়েন্ট**, এই পাঠে আপনি শিখবেন কীভাবে এমন একটি ক্লায়েন্ট লেখা যায় যা আপনার সার্ভারের সাথে সংযুক্ত হতে পারে, [পাঠটি দেখুন](02-client/README.md)

- **3 LLM সহ ক্লায়েন্ট**, ক্লায়েন্ট লেখার আরও উন্নত উপায় হল এতে একটি LLM যোগ করা যাতে এটি আপনার সার্ভারের সাথে "আলোচনা" করে কী করতে হবে, [পাঠটি দেখুন](03-llm-client/README.md)

- **4 ভিজুয়াল স্টুডিও কোডে সার্ভার GitHub Copilot এজেন্ট মোড ব্যবহার**। এখানে আমরা MCP সার্ভার ভিজুয়াল স্টুডিও কোড থেকে চালানোর পথ দেখব, [পাঠটি দেখুন](04-vscode/README.md)

- **5 stdio ট্রান্সপোর্ট সার্ভার** stdio ট্রান্সপোর্ট স্থানীয় MCP সার্ভার-টু-ক্লায়েন্ট যোগাযোগের জন্য সুপারিশকৃত স্ট্যান্ডার্ড, যা নিরাপদ subprocess-ভিত্তিক যোগাযোগ এবং বিল্ট-ইন প্রক্রিয়া বিচ্ছিন্নতা প্রদান করে [পাঠটি দেখুন](05-stdio-server/README.md)

- **6 MCP এর মাধ্যমে HTTP স্ট্রিমিং (স্ট্রিমেবল HTTP)**। আধুনিক HTTP স্ট্রিমিং ট্রান্সপোর্ট সম্পর্কে জানুন (দূরবর্তী MCP সার্ভারের জন্য সুপারিশকৃত পদ্ধতি [MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http) অনুযায়ী), অগ্রগতি বিজ্ঞপ্তি, এবং কীভাবে স্ট্রিমেবল HTTP ব্যবহার করে স্কেলযোগ্য, রিয়েল-টাইম MCP সার্ভার এবং ক্লায়েন্ট তৈরি করবেন। [পাঠটি দেখুন](06-http-streaming/README.md)

- **7 VSCode এর জন্য AI Toolkit ব্যবহার** MCP ক্লায়েন্ট এবং সার্ভারগুলি ব্যবহারের জন্য এবং পরীক্ষার জন্য [পাঠটি দেখুন](07-aitk/README.md)

- **8 পরীক্ষণ**। এখানে আমরা বিশেষভাবে ফোকাস করব কীভাবে বিভিন্ন উপায়ে আমাদের সার্ভার এবং ক্লায়েন্ট পরীক্ষা করা যায়, [পাঠটি দেখুন](08-testing/README.md)

- **9 ডিপ্লয়মেন্ট**। এই অধ্যায়ে MCP সমাধানগুলি ডিপ্লয় করার বিভিন্ন উপায় দেখানো হবে, [পাঠটি দেখুন](09-deployment/README.md)

- **10 উন্নত সার্ভার ব্যবহার**। এই অধ্যায়ে উন্নত সার্ভার ব্যবহারের বিষয়বস্তু রয়েছে, [পাঠটি দেখুন](./10-advanced/README.md)

- **11 অথেন্টিকেশন**। এই অধ্যায়ে সহজ অথেন্টিকেশন যোগ করার পদ্ধতি অন্তর্ভুক্ত, Basic Auth থেকে JWT এবং RBAC ব্যবহারে। আপনাকে এখানে শুরু করতে উৎসাহিত করা হয়েছে, তারপর অধ্যায় ৫ এর উন্নত বিষয় দেখুন এবং অধ্যায় ২ এর সুপারিশ অনুযায়ী অতিরিক্ত সুরক্ষা শক্তিশালীকরণ করুন, [পাঠটি দেখুন](./11-simple-auth/README.md)

- **12 MCP হোস্টস**। জনপ্রিয় MCP হোস্ট ক্লায়েন্ট যেমন Claude Desktop, Cursor, Cline, এবং Windsurf কনফিগার এবং ব্যবহার করুন। ট্রান্সপোর্ট প্রকার এবং সমস্যা সমাধান শিখুন, [পাঠটি দেখুন](./12-mcp-hosts/README.md)

- **13 MCP ইনস্পেক্টর**। MCP ইনস্পেক্টর টুল ব্যবহার করে আপনার MCP সার্ভারগুলি ইন্টারেকটিভভাবে ডিবাগ এবং পরীক্ষা করুন। সমস্যা নির্ণয় সরঞ্জাম, রিসোর্স এবং প্রোটোকল মেসেজ শিখুন, [পাঠটি দেখুন](./13-mcp-inspector/README.md)

- **14 স্যাম্পলিং**। MCP সার্ভার তৈরি করুন যা MCP ক্লায়েন্টদের সাথে LLM সম্পর্কিত কাজগুলিতে সহযোগিতা করে (2026-07-28 রিলিজ প্রার্থী থেকে প্রত্যাহৃত; 2025-11-25 এর জন্য এখনও বৈধ)। [পাঠটি দেখুন](./14-sampling/README.md)

- **15 MCP অ্যাপস**। MCP সার্ভার তৈরি করুন যা UI নির্দেশনাসহ উত্তর দেয়, [পাঠটি দেখুন](./15-mcp-apps/README.md)

মডেল কনটেক্সট প্রোটোকল (MCP) একটি মুক্ত প্রোটোকল যা অ্যাপ্লিকেশনগুলোকে LLM-এর জন্য প্রাসঙ্গিক তথ্য প্রদান করার জন্য স্ট্যান্ডার্ড করে তোলে। MCP কে AI অ্যাপ্লিকেশনের জন্য USB-C পোর্টের মত ভাবুন - এটি AI মডেলকে বিভিন্ন ডেটা উৎস এবং সরঞ্জামের সাথে সংযুক্ত করার একটি মানসম্মত উপায় প্রদান করে।

## শেখার লক্ষ্যসমূহ

এই পাঠ শেষ হলে আপনি সক্ষম হবেন:

- C#, Java, Python, TypeScript, এবং JavaScript এ MCP এর জন্য ডেভেলপমেন্ট পরিবেশ তৈরি করতে
- কাস্টম ফিচার (রিসোর্স, প্রম্পট, এবং টুলস) সহ মৌলিক MCP সার্ভার তৈরি এবং ডিপ্লয় করতে
- MCP সার্ভারের সাথে সংযুক্ত হোস্ট অ্যাপ্লিকেশন তৈরি করতে
- MCP বাস্তবায়ন পরীক্ষা ও ডিবাগ করতে
- সাধারণ সেটআপ চ্যালেঞ্জ এবং তাদের সমাধান বুঝতে
- আপনার MCP বাস্তবায়ন জনপ্রিয় LLM সার্ভিসে সংযুক্ত করতে

## আপনার MCP পরিবেশ সেটআপ করা

MCP নিয়ে কাজ শুরু করার আগে, আপনার ডেভেলপমেন্ট পরিবেশ প্রস্তুত করা এবং মৌলিক কর্মপ্রবাহ বোঝা গুরুত্বপূর্ণ। এই অংশ আপনাকে MCP সঙ্গে একটি মসৃণ শুরু নিশ্চিত করতে প্রথম সেটআপ ধাপগুলোতে দিকনির্দেশনা দেবে।

### পূর্বশর্ত

MCP ডেভেলপমেন্টে ঝাঁপ দেওয়ার আগে নিশ্চিত করুন আপনার কাছে আছে:

- **ডেভেলপমেন্ট পরিবেশ**: আপনার নির্বাচিত ভাষার জন্য (C#, Java, Python, TypeScript, অথবা JavaScript)
- **IDE/এডিটর**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm, অথবা যেকোন আধুনিক কোড এডিটর
- **প্যাকেজ ম্যানেজার**: NuGet, Maven/Gradle, pip, অথবা npm/yarn
- **API কী**: আপনার হোস্ট অ্যাপ্লিকেশনগুলিতে যেসব AI পরিষেবা ব্যাবহার করবেন সেগুলোর জন্য


### অফিসিয়াল SDK গুলো

আগামী অধ্যায়গুলোতে আপনি Python, TypeScript, Java এবং .NET ব্যবহার করে নির্মিত সমাধান দেখতে পাবেন। এখানে সব অফিসিয়ালভাবে সমর্থিত SDK রয়েছে।

MCP বিভিন্ন ভাষার জন্য অফিসিয়াল SDK প্রদান করে ([MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/) অনুযায়ী):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - Microsoft-এর সাথে সহযোগিতায় রক্ষণাবেক্ষণ
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI-এর সাথে সহযোগিতায় রক্ষণাবেক্ষণ
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - অফিসিয়াল TypeScript ইমপ্লিমেন্টেশন
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - অফিসিয়াল Python ইমপ্লিমেন্টেশন (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - অফিসিয়াল Kotlin ইমপ্লিমেন্টেশন
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI-এর সাথে সহযোগিতায় রক্ষণাবেক্ষণ
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - অফিসিয়াল Rust ইমপ্লিমেন্টেশন
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - অফিসিয়াল Go ইমপ্লিমেন্টেশন

## মূল উপসংহার

- MCP ডেভেলপমেন্ট পরিবেশ সেটআপ ভাষাভিত্তিক SDK দ্বারা সরল
- MCP সার্ভার তৈরি করা মানে পরিষ্কার স্কিমাস সহ টুল তৈরি ও নিবন্ধন করা
- MCP ক্লায়েন্ট সার্ভার ও মডেলগুলোর সাথে সংযোগ করে সম্প্রসারিত সক্ষমতা ব্যবহার করে
- পরীক্ষণ ও ডিবাগিং গুরুত্বপূর্ণ MCP বাস্তবায়নের নির্ভরযোগ্যতার জন্য
- ডিপ্লয়মেন্ট বিকল্প স্থানীয় ডেভেলপমেন্ট থেকে ক্লাউড-ভিত্তিক সমাধান পর্যন্ত বিস্তৃত
