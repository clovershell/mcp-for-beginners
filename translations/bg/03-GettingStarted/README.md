Този раздел се състои от няколко урока:

- **1 Вашият първи сървър**, в този първи урок ще научите как да създадете своя първи сървър и да го инспектирате с инструмента инспектор, ценен начин за тестване и отстраняване на грешки на вашия сървър, [към урока](01-first-server/README.md)

- **2 Клиент**, в този урок ще научите как да напишете клиент, който може да се свърже към вашия сървър, [към урока](02-client/README.md)

- **3 Клиент с LLM**, още по-добър начин за писане на клиент е чрез добавяне на LLM, така че той може да "преговаря" с вашия сървър какво да прави, [към урока](03-llm-client/README.md)

- **4 Използване на режим GitHub Copilot Agent за сървър в Visual Studio Code**. Тук разглеждаме как да стартираме нашия MCP сървър от Visual Studio Code, [към урока](04-vscode/README.md)

- **5 stdio Транспортен сървър** stdio транспортът е препоръчителният стандарт за локална комуникация между MCP сървър и клиент, осигуряващ сигурна комуникация на подпрограми с вградена изолация на процесите [към урока](05-stdio-server/README.md)

- **6 HTTP Стрийминг с MCP (Streamable HTTP)**. Научете за модерния HTTP стрийминг транспорт (препоръчителният подход за отдалечени MCP сървъри според [Спецификация MCP 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http)), известия за напредък и как да реализирате мащабируеми, в реално време MCP сървъри и клиенти чрез Streamable HTTP. [към урока](06-http-streaming/README.md)

- **7 Използване на AI Toolkit за VSCode** за консумация и тест на ваши MCP клиенти и сървъри [към урока](07-aitk/README.md)

- **8 Тестване**. Тук ще се фокусираме специално върху различни начини за тестване на нашия сървър и клиент, [към урока](08-testing/README.md)

- **9 Деплоймънт**. Тази глава разглежда различни начини за разполагане на вашите MCP решения, [към урока](09-deployment/README.md)

- **10 Разширено използване на сървър**. Тази глава покрива усъвършенствано използване на сървъра, [към урока](./10-advanced/README.md)

- **11 Аутентикация**. Тази глава обяснява как да добавите проста аутентикация, от Basic Auth до използване на JWT и RBAC. Препоръчва се да започнете тук и после да разгледате разширените теми в Глава 5 и да направите допълнително затягане на сигурността чрез препоръки в Глава 2, [към урока](./11-simple-auth/README.md)

- **12 MCP Хостове**. Конфигурирайте и използвайте популярни MCP клиентски хостове като Claude Desktop, Cursor, Cline и Windsurf. Научете типовете транспорт и отстраняване на проблеми, [към урока](./12-mcp-hosts/README.md)

- **13 MCP Инспектор**. Отстранявайте грешки и тествайте вашите MCP сървъри интерактивно с инструмента MCP Inspector. Научете как да отстранявате проблеми с инструменти, ресурси и протоколни съобщения, [към урока](./13-mcp-inspector/README.md)

- **14 Семплиране**. Създавайте MCP сървъри, които си сътрудничат с MCP клиенти по задачи, свързани с LLM (отпаднало в кандидат за издание `2026-07-28`; все още валидно за `2025-11-25`). [към урока](./14-sampling/README.md)

- **15 MCP Приложения**. Изградете MCP сървъри, които също отговарят с UI инструкции, [към урока](./15-mcp-apps/README.md)

Протоколът за контекст на модела (MCP) е отворен протокол, който стандартизира как приложенията предоставят контекст на LLM. Представете си MCP като USB-C порт за AI приложения - той осигурява стандартизиран начин за свързване на AI модели с различни източници на данни и инструменти.

## Цели на обучението

Края на този урок ще ви позволи да:

- Настроите развойна среда за MCP на C#, Java, Python, TypeScript и JavaScript
- Изградите и разгърнете базови MCP сървъри с персонализирани функции (ресурси, подканящи съобщения и инструменти)
- Създавате хост приложения, които се свързват с MCP сървъри
- Тествате и отстранявате грешки в реализации на MCP
- Разбирате често срещаните предизвикателства при настройка и техните решения
- Свържете вашите MCP реализации с популярни LLM услуги

## Настройване на вашата MCP среда

Преди да започнете работа с MCP, е важно да подготвите развойната си среда и да разберете основния работен процес. Този раздел ще ви води през първоначалните стъпки за настройка, за да осигури плавен старт с MCP.

### Изисквания

Преди да се потопите в разработката с MCP, уверете се, че имате:

- **Развойна среда**: За избрания от вас език (C#, Java, Python, TypeScript или JavaScript)
- **IDE/Редактор**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm или всеки модерен код редактор
- **Пакетни мениджъри**: NuGet, Maven/Gradle, pip или npm/yarn
- **API ключове**: За всички AI услуги, които планирате да използвате в хост приложенията си


### Официални SDK-та

В следващите глави ще видите решения, изградени с Python, TypeScript, Java и .NET. Ето всички официално поддържани SDK-та.

MCP предлага официални SDK-та за няколко езика (съобразени с [Спецификация MCP 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/)):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - Поддържан в сътрудничество с Microsoft
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Поддържан в сътрудничество с Spring AI
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - Официалната TypeScript реализация
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - Официалната Python реализация (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - Официалната Kotlin реализация
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Поддържан в сътрудничество с Loopwork AI
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - Официалната Rust реализация
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - Официалната Go реализация

## Ключови изводи

- Настройването на развойна среда за MCP е лесно с езиково-специфични SDK-та
- Изграждането на MCP сървъри включва създаването и регистрирането на инструменти с ясни схеми
- MCP клиентите се свързват със сървъри и модели, за да използват разширени възможности
- Тестването и отстраняването на грешки са от съществено значение за надеждни реализации на MCP
- Опциите за разгръщане варират от локална разработка до решения базирани в облак
