## ಪ್ರಾರಂಭ  

[![ನಿಮ್ಮ ಮೊದಲ MCP ಸರ್ವರ್ ರಚಿಸಿ](../../../translated_images/kn/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(ಈ ಪಾಠದ ವಿಡಿಯೋ ನೋಡಲು ಮೇಲಿನ ಚಿತ್ರವನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ)_

ಈ ಭಾಗದಲ್ಲಿ ಅನೇಕ ಪಾಠಗಳಿವೆ:

- **1 ನಿಮ್ಮ ಮೊದಲ ಸರ್ವರ್**, ಈ ಮೊದಲ ಪಾಠದಲ್ಲಿ, ನೀವು ನಿಮ್ಮ ಮೊದಲ ಸರ್ವರ್ ರಚಿಸುವುದು ಮತ್ತು ಅದನ್ನು ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಸಾಧನದಿಂದ ಪರಿಶೀಲಿಸುವುದನ್ನು ಕಲಿಯುತ್ತೀರಿ, ಇದು ನಿಮ್ಮ ಸರ್ವರ್ ಅನ್ನು ಪರೀಕ್ಷಿಸಲು ಮತ್ತು ಡಿಬಗ್ ಮಾಡಲು ಒಳ್ಳೆಯ ವಿಧಾನವಾಗಿದೆ, [ಪಾಠಕ್ಕೆ](01-first-server/README.md)

- **2 ಕ್ಲಯಂಟ್**, ಈ ಪಾಠದಲ್ಲಿ, ನೀವು ನಿಮ್ಮ ಸರ್ವರ್ ಜೊತೆಗೆ ಸಂಪರ್ಕ ಮಾಡಬಲ್ಲ ಕ್ಲಯಂಟ್ ಬರೆಯುವುದು ಹೇಗೆ ಎಂಬುದನ್ನು ಕಲಿಯುತ್ತೀರಿ, [ಪಾಠಕ್ಕೆ](02-client/README.md)

- **3 LLM ಹೊಂದಿರುವ ಕ್ಲಯಂಟ್**, ಕ್ಲಯಂಟ್ ಬರೆಯುವ ಇನ್ನೂ ಉತ್ತಮ ವಿಧಾನವೆಂದರೆ ಅದರೊಂದಿಗೆ LLM ಅನ್ನು ಸೇರ್ಪಡೆ ಮಾಡುವುದು, ಅದು ನಿಮ್ಮ ಸರ್ವರ್ ಜೊತೆ "ಚರ್ಚೆ" ಮಾಡಿ ಏನು ಮಾಡಬೇಕೆಂದು ನಿರ್ಧರಿಸಬಹುದು, [ಪಾಠಕ್ಕೆ](03-llm-client/README.md)

- **4 Visual Studio Code ನಲ್ಲಿ GitHub Copilot ಏಜೆಂಟ್ ಮೋಡ್ ಬಳಸಿ ಸರ್ವರ್ ಉಪಯೋಗಿಸುವುದು**. ಇಲ್ಲಿ ನಾವು Visual Studio Code ಒಳಗಿಂದ MCP ಸರ್ವರ್ ನಡೆಸುವುದನ್ನು ನೋಡಿ, [ಪಾಠಕ್ಕೆ](04-vscode/README.md)

- **5 stdio ಸಾರಿಗೆ ಸರ್ವರ್** stdio ಸಾರಿಗೆ ಸ್ಥಳೀಯ MCP ಸರ್ವರ್-ಕ್ಲಯಂಟ್ ಸಂವಹನಕ್ಕಾಗಿ ಶಿಫಾರಸುಗೊಂಡ ಮಾನ್ದಂಡ, ಇದು ಸುರಕ್ಷಿತ ಉಪಪ್ರಕ್ರಿಯೆ ಆಧಾರಿತ ಸಂವಹನ ಮತ್ತು ಪೃಕ್ರಿಯೆಯ ವಿಶ್ಲೇಷಣೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ [ಪಾಠಕ್ಕೆ](05-stdio-server/README.md)

- **6 MCP ಸಹಿತ HTTP ಸ್ಟ್ರೀಮಿಂಗ್ (ಸ್ಟ್ರೀಮಾನ್ಬಲ್ HTTP)**. ಆಧುನಿಕ HTTP ಸ್ಟ್ರೀಮಿಂಗ್ ಸಾರಿಗೆ ಕುರಿತು ತಿಳಿದುಕೊಳ್ಳಿ (ವಿದೇಶೀ MCP ಸರ್ವರ್‌ಗಳಿಗೆ ಶಿಫಾರಸು ಮಾಡುವ ವಿಧಾನ [MCP ವಿಶೇಷಣ 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http)), ಪ್ರಗತಿ ತಿಳಿಸುವುದು, ಮತ್ತು ಸ್ಟ್ರೀಮಾನ್ಬಲ್ HTTP ಬಳಸಿ ವ್ಯವಹರಿಸಬಹುದಾದ, ನೈಜ ಸಮಯ MCP ಸರ್ವರ್ಗಳು ಮತ್ತು ಕ್ಲಯಂಟ್‌ಗಳನ್ನು ಹೇಗೆ ಜಾರಿ ಮಾಡುವುದು ಎಂಬುದನ್ನು ಕಲಿಯಿರಿ. [ಪಾಠಕ್ಕೆ](06-http-streaming/README.md)

- **7 VSCode ಗಾಗಿ AI ಉಪಕರಣಕೋಶವನ್ನು ಉಪಯೋಗಿಸುವುದು** ನಿಮ್ಮ MCP ಕ್ಲಯಂಟ್ ಮತ್ತು ಸರ್ವರ್ಗಳನ್ನು ಪರೀಕ್ಷಿಸಲು, [ಪಾಠಕ್ಕೆ](07-aitk/README.md)

- **8 ಪರೀಕ್ಷೆ**. ಇಲ್ಲಿ ನಾವು ವಿಶೇಷವಾಗಿ ನಮ್ಮ ಸರ್ವರ್ ಮತ್ತು ಕ್ಲಯಂಟ್ ಅನ್ನು ವಿವಿಧ ರೀತಿಯಲ್ಲಿ ಪರೀಕ್ಷಿಸುವುದರ ಬಗ್ಗೆ ಗಮನಹರಿಸುವೆವು, [ಪಾಠಕ್ಕೆ](08-testing/README.md)

- **9 ನಿಯೋಜನೆ**. ಈ ಅಧ್ಯಾಯದಲ್ಲಿ ನಿಮ್ಮ MCP ಪರಿಹಾರಗಳನ್ನು ನಿಯೋಜಿಸುವ ವಿವಿಧ ವಿಧಾನಗಳನ್ನು ನೋಡಿಕೊಳ್ಳಲಾಗುತ್ತದೆ, [ಪಾಠಕ್ಕೆ](09-deployment/README.md)

- **10 ಉನ್ನತ ಸರ್ವರ್ ಬಳಕೆ**. ಈ ಅಧ್ಯಾಯವು ಉನ್ನತ ಸರ್ವರ್ ಬಳಕೆಯನ್ನು ಒಳಗೊಂಡಿದೆ, [ಪಾಠಕ್ಕೆ](./10-advanced/README.md)

- **11 ಪ್ರಾಮಾಣೀಕರಣ**. ಈ ಅಧ್ಯಾಯವು ಸರಳ ಪ್ರಾಮಾಣೀಕರಣವನ್ನು ಸೇರಿಸುವ ಬಗ್ಗೆ (ಬೇಸಿಕ್ ಆಥ್ ನಿಂದ JWT ಮತ್ತು RBAC ಬಳಕೆಗಡೆಗೆ) ಕವರ್ ಮಾಡುತ್ತದೆ. ನೀವು ಇಲ್ಲಿ ಪ್ರಾರಂಭಿಸಲು ಪ್ರೋತ್ಸಾಹಿತರಾಗಿದ್ದು ನಂತರ ಅಧ್ಯಾಯ 5 ರಲ್ಲಿ ಉನ್ನತ ವಿಷಯಗಳನ್ನು ನೋಡಿ, ಮತ್ತು ಅಧ್ಯಾಯ 2 ರಲ್ಲಿನ ಶಿಫಾರಸುಗಳ ಅಡಿಯಲ್ಲಿ ಹೆಚ್ಚುವರಿ ಭದ್ರತಾ ಸೂಕ್ತತೆ ಮಾಡಲು ಸಹಾಯಮಾಡುವುದು, [ಪಾಠಕ್ಕೆ](./11-simple-auth/README.md)

- **12 MCP ಹೋಸ್ಟ್‌ಗಳು**. ಪ್ರಮುಖ MCP ಹೋಸ್ಟ್ ಕ್ಲಯಂಟ್‌ಗಳು ಕೇಲಾಡ್ ಡೆಸ್ಕ್‌ಟಾಪ್, ಕರ್ಸರ್, ಕ್ಲೈನ್ ಮತ್ತು ವಿಂಡ್ಸರ್ಫ್ ಸೇರಿ ಹೇಗೆ ಸಂರಚಿಸಿ ಮತ್ತು ಉಪಯೋಗಿಸುವುದು, ಸಂವಹನದ ವಿಧಗಳು ಮತ್ತು ಸಮಸ್ಯೆ ಪರಿಹಾರಗಳನ್ನು ಕಲಿಯಿರಿ, [ಪಾಠಕ್ಕೆ](./12-mcp-hosts/README.md)

- **13 MCP ಇನ್ಸ್ಪೆಕ್ಟರ್**. MCP ಸರ್ವರ್ಗಳನ್ನು ಸಂವಾದಾತ್ಮಕವಾಗಿ ಡಿಬಗ್ ಮತ್ತು ಪರೀಕ್ಷಿಸಲು MCP ಇನ್ಸ್ಪೆಕ್ಟರ್ ಉಪಕರಣವನ್ನು ಬಳಸಿ. ಸಾಧನಗಳು, ಸಂಪನ್ಮೂಲಗಳು ಮತ್ತು ಪ್ರೋಟೋಕಾಲ್ ಸಂದೇಶಗಳ ಸಮಸ್ಯೆ ಪರಿಹಾರ ಕಲಿಯಿರಿ, [ಪಾಠಕ್ಕೆ](./13-mcp-inspector/README.md)

- **14 ಸಮೀಕ್ಷೆ**. LLM ಸಂಬಂಧಿತ ಕಾರ್ಯಗಳಲ್ಲಿ MCP ಕ್ಲಯಂಟ್‌ಗಳೊಂದಿಗೆ ಸಹಕಾರ ಮಾಡುವ MCP ಸರ್ವರ್ಗಳನ್ನು ರಚಿಸಿ (`2026-07-28` ಬಿಡುಗಡೆ ಅಭ್ಯರ್ಥಿಯಲ್ಲಿ ಬಣ್ಣಿಸಲು; `2025-11-25` ರಲ್ಲಿಯೂ ಯಶಸ್ವಿ), [ಪಾಠಕ್ಕೆ](./14-sampling/README.md)

- **15 MCP ಅಪ್ಲಿಕೇಶನ್‌ಗಳು**. UI ಸೂಚನೆಗಳನ್ನು ಉತ್ತರಿಸುವ MCP ಸರ್ವರ್ಗಳನ್ನು ರಚಿಸಿ, [ಪಾಠಕ್ಕೆ](./15-mcp-apps/README.md)

ಮಾದರಿ ಸಂಧರ್ಭ ಪ್ರೋಟೋಕಾಲ್ (MCP) ಒಂದು ತೆರೆಯಲಾದ ಪ್ರೋಟೋಕಾಲ್ ಆಗಿದ್ದು, ಅಪ್ಲಿಕೇಶನ್‌ಗಳು LLM ಗೆ ಸಂಧರ್ಭವನ್ನು ಒದಗಿಸುವ ವಿಧಾನವನ್ನು ಮಾನದಂಡಗೊಳಿಸುತ್ತದೆ. MCP ಅನ್ನು AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗಾಗಿ USB-C ಪೋರ್ಟ್ ಎಂದು ಭಾವಿಸಿ - ಇದು ವಿವಿಧ ಡೇಟಾ ಮೂಲಗಳು ಮತ್ತು ಸಾಧನಗಳಿಗೆ AI ಮಾದರಿಗಳನ್ನು ಸಂಧಿಸಲು ಮಾನದಂಡಿತ ವಿಧಾನವನ್ನು ಒದಗಿಸುತ್ತದೆ.

## ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳು

ಈ ಪಾಠದ ಕೊನೆಯಲ್ಲಿ ನೀವು ಸಮರ್ಥರಾಗಿರುತ್ತೀರಿ:

- C#, Java, Python, TypeScript ಮತ್ತು JavaScript ನಲ್ಲಿ MCP ಅಭಿವೃದ್ಧಿ ವಾತಾವರಣಗಳನ್ನು ಹೊಂದಿಸುವುದು
- ಕಸ್ಟಮ್ ವೈಶಿಷ್ಟ್ಯಗಳೊಂದಿಗೆ ಮೂಲ MCP ಸರ್ವರ್ಗಳನ್ನು ನಿರ್ಮಿಸಿ ಮತ್ತು ನಿಯೋಜಿಸುವುದು (ಸಂಪನ್ಮೂಲಗಳು, ಪ್ರಾಂಪ್ಟ್ಗಳು, ಮತ್ತು ಸಾಧನಗಳು)
- MCP ಸರ್ವರ್ಗಳಿಗೆ ಸಂಪರ್ಕಿಸುವ ಹೋಸ್ಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ರಚಿಸುವುದು
- MCP ಅನುಷ್ಠಾನಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ ಮತ್ತು ಡಿಬಗ್ ಮಾಡುವುದು
- ಸಾಮಾನ್ಯ ಸ್ಥಾಪನೆ ಸವಾಲುಗಳು ಮತ್ತು ಅವುಗಳ ಪರಿಹಾರಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು
- ನಿಮ್ಮ MCP ಅನುಷ್ಠಾನಗಳನ್ನು ಜನಪ್ರಿಯ LLM ಸೇವೆಗಳಿಗೆ ಸಂಪರ್ಕಿಸುವುದು

## ನಿಮ್ಮ MCP ವಾತಾವರಣವನ್ನು ಹೊಂದಿಸುವುದು

MCP ಜೊತೆ ಕೆಲಸ ಆರಂಭಿಸುವ ಮೊದಲು, ನಿಮ್ಮ ಅಭಿವೃದ್ಧಿ ವಾತಾವರಣವನ್ನು ಸಜ್ಜುಗೊಳಿಸುವುದು ಮತ್ತು ಮೂಲ ಕಾರ್ಯಪ್ರವಾಹವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಮಹತ್ವಾದಾಯಕ. ಈ ವಿಭಾಗವು MCP ಜೊತೆ ಸುಗಮ ಪ್ರಾರಂಭಕ್ಕೆ ಪ್ರಾಥಮಿಕ ಸ್ಥಾಪನೆ ಹಂತಗಳನ್ನು ಮಾರ್ಗದರ್ಶಿಸುತ್ತದೆ.

### ಪೂರ್ವಅವಶ್ಯಕತೆಗಳು

MCP ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ ತೊಡಗಿಸುವ ಮುನ್ನ, ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ ಬಳಕೆಗಾರನಿಗೆ:

- **ಅಭಿವೃದ್ಧಿ ವಾತಾವರಣ**: ನಿಮ್ಮ ಆಯ್ದ ಭಾಷೆಗೆ (C#, Java, Python, TypeScript ಅಥವಾ JavaScript)
- **IDE/ಸಂಪಾದಕ**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm ಅಥವಾ ಯಾವುದೇ ಆಧುನಿಕ ಕೋಡ್ ಸಂಪಾದಕ
- **ಪ್ಯಾಕೇಜ್ ನಿರ್ವಾಹಕರು**: NuGet, Maven/Gradle, pip, ಅಥವಾ npm/yarn
- **API ಕೀಗಳು**: ನಿಮ್ಮ ಹೋಸ್ಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಲ್ಲಿ ಉಪಯೋಗಿಸಲು ಯೋಜಿಸಿರುವ ಯಾವುದೇ AI ಸೇವೆಗಳಿಗಾಗಿ


### ಅಧಿಕೃತ SDKಗಳು

ಮುಂದಿನ ಅಧ್ಯಾಯಗಳಲ್ಲಿ ನೀವು Python, TypeScript, Java ಮತ್ತು .NET ಬಳಸಿ ನಿರ್ಮಿಸಿದ ಪರಿಹಾರಗಳನ್ನು ನೋಡುತ್ತೀರಿ. ಇವು ಎಲ್ಲಾ ಅಧಿಕೃತವಾಗಿ ಬೆಂಬಲಿಸಲ್ಪಟ್ಟ SDKಗಳು.

MCP ಹಲವು ಭಾಷೆಗಳಿಗಾಗಿ ಅಧಿಕೃತ SDKಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ ([MCP ವಿಶೇಷಣ 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/)):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - Microsoft ಜೊತೆಗೆ ಸಹಯೋಗದಲ್ಲಿ ನಿರ್ವಹಣೆ
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI ಜೊತೆ ಸಹಯೋಗದಲ್ಲಿ ನಿರ್ವಹಣೆ
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - ಅಧಿಕೃತ TypeScript ಅನುಷ್ಠಾನ
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - ಅಧಿಕೃತ Python ಅನುಷ್ಠಾನ (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - ಅಧಿಕೃತ Kotlin ಅನುಷ್ಠಾನ
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI ಜೊತೆ ಸಹಯೋಗದಲ್ಲಿ ನಿರ್ವಹಣೆ
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - ಅಧಿಕೃತ Rust ಅನುಷ್ಠಾನ
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - ಅಧಿಕೃತ Go ಅನುಷ್ಠಾನ

## ಪ್ರಮುಖ ತರಬೇತುಗ್ರಹಿಕೆಗಳು

- ಭಾಷಾ ವಿಶಿಷ್ಟ SDKಗಳೊಂದಿಗೆ MCP ಅಭಿವೃದ್ಧಿ ವಾತಾವರಣವನ್ನು ಹೊಂದಿಸುವುದು ಸರಳ
- MCP ಸರ್ವರ್ಗಳ ನಿರ್ಮಾಣಕ್ಕೆ ಸ್ಪಷ್ಟ ಸ್ವರೂಪದ ಸಾಧನಗಳನ್ನು ರಚಿಸಿ ಮತ್ತು ನೋಂದಾಯಿಸುವುದು ಸೇರಿದೆ
- MCP ಕ್ಲಯಂಟ್‌ಗಳು ಸರ್ವರ್ಗಳು ಮತ್ತು ಮಾದರಿಗಳೊಂದಿಗೆ ಸಂಪರ್ಕಿಸಿ ವಿಸ್ತೃತ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಉಪಯೋಗಿಸುತ್ತವೆ
- ವಿಶ್ವಾಸಾರ್ಹ MCP ಅನುಷ್ಠಾನಗಳಿಗೆ ಪರೀಕ್ಷೆ ಮತ್ತು ಡಿಬಗ್ ಅವಶ್ಯಕ
- ನಿಯೋಜನೆ ಆಯ್ಕೆಗಳು ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿಯಿಂದ ಮೇಘಾಧಾರಿತ ಪರಿಹಾರಗಳವರೆಗೆ ಇರುತ್ತವೆ
