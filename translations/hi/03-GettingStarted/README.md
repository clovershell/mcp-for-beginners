[![अपना पहला MCP सर्वर बनाएं](../../../translated_images/hi/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(इस पाठ के वीडियो को देखने के लिए ऊपर दी गई छवि पर क्लिक करें)_

इस अनुभाग में कई पाठ शामिल हैं:

- **1 आपका पहला सर्वर**, इस पहले पाठ में, आप सीखेंगे कि कैसे अपना पहला सर्वर बनाएं और निरीक्षक उपकरण के साथ इसे निरीक्षण करें, जो आपके सर्वर का परीक्षण और डीबग करने का एक मूल्यवान तरीका है, [पाठ के लिए](01-first-server/README.md)

- **2 क्लाइंट**, इस पाठ में, आप सीखेंगे कि कैसे एक क्लाइंट लिखें जो आपके सर्वर से कनेक्ट हो सके, [पाठ के लिए](02-client/README.md)

- **3 LLM के साथ क्लाइंट**, क्लाइंट लिखने का एक और भी बेहतर तरीका है कि उसमें एक LLM जोड़ना ताकि वह "बातचीत" कर सके कि सर्वर पर क्या करना है, [पाठ के लिए](03-llm-client/README.md)

- **4 GitHub Copilot एजेंट मोड को Visual Studio Code में सर्वर का उपयोग करना**। यहाँ हम Visual Studio Code के भीतर से अपना MCP सर्वर चलाने पर ध्यान दे रहे हैं, [पाठ के लिए](04-vscode/README.md)

- **5 stdio ट्रांसपोर्ट सर्वर** stdio ट्रांसपोर्ट स्थानीय MCP सर्वर-से-क्लाइंट संचार के लिए अनुशंसित मानक है, जो सुरक्षित subprocess-आधारित संचार को अंतर्निहित प्रक्रिया पृथक्करण के साथ प्रदान करता है [पाठ के लिए](05-stdio-server/README.md)

- **6 MCP के साथ HTTP स्ट्रीमिंग (Streamable HTTP)**। आधुनिक HTTP स्ट्रीमिंग ट्रांसपोर्ट के बारे में जानें (दूरस्थ MCP सर्वरों के लिए अनुशंसित तरीका [MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http) के अनुसार), प्रगति सूचनाएं, और कैसे Streamable HTTP का उपयोग करके स्केलेबल, वास्तविक समय MCP सर्वर और क्लाइंट बनाएँ। [पाठ के लिए](06-http-streaming/README.md)

- **7 VSCode के लिए AI टूलकिट का उपयोग** अपने MCP क्लाइंट और सर्वर का उपयोग और परीक्षण करने के लिए [पाठ के लिए](07-aitk/README.md)

- **8 परीक्षण**। यहाँ हम विशेष रूप से यह देखेंगे कि हम अपने सर्वर और क्लाइंट का विभिन्न तरीकों से परीक्षण कैसे कर सकते हैं, [पाठ के लिए](08-testing/README.md)

- **9 परिनियोजन**। यह अध्याय आपके MCP समाधान को तैनात करने के विभिन्न तरीकों पर नजर डालता है, [पाठ के लिए](09-deployment/README.md)

- **10 उन्नत सर्वर उपयोग**। यह अध्याय उन्नत सर्वर उपयोग को कवर करता है, [पाठ के लिए](./10-advanced/README.md)

- **11 प्रमाणीकरण**। यह अध्याय सरल प्रमाणीकरण जोड़ने को कवर करता है, बेसिक ऑथ से लेकर JWT और RBAC के उपयोग तक। आपको इसे यहाँ शुरू करने और फिर अध्याय 5 के उन्नत विषयों और अध्याय 2 के सुझावों द्वारा अतिरिक्त सुरक्षा सुदृढ़ीकरण करने के लिए प्रोत्साहित किया जाता है, [पाठ के लिए](./11-simple-auth/README.md)

- **12 MCP होस्ट्स**। लोकप्रिय MCP होस्ट क्लाइंट्स जैसे Claude Desktop, Cursor, Cline, और Windsurf को कॉन्फ़िगर करें और उपयोग करें। ट्रांसपोर्ट प्रकार और समस्या निवारण के बारे में जानें, [पाठ के लिए](./12-mcp-hosts/README.md)

- **13 MCP निरीक्षक**। अपने MCP सर्वरों का इंटरैक्टिव रूप से डीबग और परीक्षण करें MCP निरीक्षक उपकरण का उपयोग करके। समस्या निवारण उपकरण, संसाधन, और प्रोटोकॉल संदेशों को सीखें, [पाठ के लिए](./13-mcp-inspector/README.md)

- **14 सैंपलिंग**। MCP सर्वर बनाएँ जो MCP क्लाइंट के साथ LLM संबंधित कार्यों पर सहयोग करते हैं ( `2026-07-28` रिलीज उम्मीदवार में अप्रचलित; फिर भी `2025-11-25` के लिए मान्य)। [पाठ के लिए](./14-sampling/README.md)

- **15 MCP ऐप्स**। ऐसे MCP सर्वर बनाएं जो UI निर्देशों के साथ भी जवाब देते हैं, [पाठ के लिए](./15-mcp-apps/README.md)

मॉडल कॉन्टेक्स्ट प्रोटोकॉल (MCP) एक खुला प्रोटोकॉल है जो निर्दिष्ट करता है कि एप्लिकेशन LLMs को संदर्भ कैसे प्रदान करते हैं। MCP को AI एप्लिकेशन के लिए एक USB-C पोर्ट के रूप में सोचें - यह AI मॉडल को विभिन्न डेटा स्रोतों और टूल्स से जोड़ने का एक मानकीकृत तरीका प्रदान करता है।

## सीखने के उद्देश्य

इस पाठ के अंत तक, आप सक्षम होंगे:

- C#, Java, Python, TypeScript, और JavaScript में MCP के लिए विकास पर्यावरण सेट अप करना
- कस्टम फीचर्स (संसाधन, प्रॉम्प्ट, और टूल्स) के साथ बुनियादी MCP सर्वर बनाना और तैनात करना
- MCP सर्वर से जुड़ने वाले होस्ट एप्लिकेशन बनाना
- MCP कार्यान्वयन का परीक्षण और डीबग करना
- सामान्य सेटअप चुनौतियों और उनके समाधान को समझना
- अपने MCP कार्यान्वयन को लोकप्रिय LLM सेवाओं से कनेक्ट करना

## अपने MCP वातावरण की सेटिंग

MCP के साथ काम शुरू करने से पहले, यह आवश्यक है कि आप अपने विकास वातावरण को तैयार करें और मूल कार्यप्रवाह को समझें। यह अनुभाग आपको प्रारंभिक सेटअप चरणों के माध्यम से मार्गदर्शन करेगा ताकि MCP के साथ सुचारु शुरुआत हो सके।

### पूर्वापेक्षाएँ

MCP विकास में कूदने से पहले, सुनिश्चित करें कि आपके पास निम्नलिखित हैं:

- **विकास वातावरण**: अपनी चुनी हुई भाषा (C#, Java, Python, TypeScript, या JavaScript) के लिए
- **IDE/संपादक**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm, या कोई आधुनिक कोड संपादक
- **पैकेज मैनेजर**: NuGet, Maven/Gradle, pip, या npm/yarn
- **API कुंजियाँ**: किसी भी AI सेवा के लिए जो आप अपने होस्ट एप्लिकेशन में उपयोग करने का योजना बना रहे हैं


### आधिकारिक SDKs

आगामी अध्यायों में आप Python, TypeScript, Java और .NET का उपयोग करके समाधान देखेंगे। यहाँ सभी आधिकारिक समर्थित SDKs हैं।

MCP कई भाषाओं के लिए आधिकारिक SDK प्रदान करता है ([MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/) के अनुरूप):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - Microsoft के सहयोग से बनाए रखा गया
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI के सहयोग से बनाए रखा गया
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - आधिकारिक TypeScript कार्यान्वयन
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - आधिकारिक Python कार्यान्वयन (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - आधिकारिक Kotlin कार्यान्वयन
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI के सहयोग से बनाए रखा गया
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - आधिकारिक Rust कार्यान्वयन
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - आधिकारिक Go कार्यान्वयन

## मुख्य बिंदु

- MCP विकास वातावरण भाषा-विशिष्ट SDK के साथ सेट करना सरल है
- MCP सर्वरों का निर्माण स्पष्ट स्कीमा के साथ टूल बनाने और पंजीकृत करने में शामिल है
- MCP क्लाइंट सर्वर और मॉडल से कनेक्ट होकर विस्तारित क्षमताओं का लाभ उठाते हैं
- विश्वसनीय MCP कार्यान्वयन के लिए परीक्षण और डीबग आवश्यक हैं
- परिनियोजन के विकल्प स्थानीय विकास से लेकर क्लाउड-आधारित समाधान तक होते हैं
