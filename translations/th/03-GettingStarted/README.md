## การเริ่มต้น  

[![สร้าง MCP Server ตัวแรกของคุณ](../../../translated_images/th/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(คลิกที่รูปภาพด้านบนเพื่อดูวิดีโอบทเรียนนี้)_

ส่วนนี้ประกอบด้วยบทเรียนหลายบท:

- **1 เซิร์ฟเวอร์ตัวแรกของคุณ** ในบทเรียนแรกนี้ คุณจะได้เรียนรู้วิธีสร้างเซิร์ฟเวอร์ตัวแรกของคุณและตรวจสอบด้วยเครื่องมือ inspector ซึ่งเป็นวิธีที่มีคุณค่าในการทดสอบและแก้ไขข้อบกพร่องเซิร์ฟเวอร์ของคุณ [ไปที่บทเรียน](01-first-server/README.md)

- **2 ไคลเอนต์** ในบทเรียนนี้ คุณจะได้เรียนรู้วิธีเขียนไคลเอนต์ที่สามารถเชื่อมต่อกับเซิร์ฟเวอร์ของคุณได้ [ไปที่บทเรียน](02-client/README.md)

- **3 ไคลเอนต์กับ LLM** วิธีที่ดียิ่งขึ้นในการเขียนไคลเอนต์คือโดยการเพิ่ม LLM เข้าไป เพื่อให้มันสามารถ "เจรจา" กับเซิร์ฟเวอร์ของคุณเกี่ยวกับสิ่งที่จะทำได้ [ไปที่บทเรียน](03-llm-client/README.md)

- **4 การใช้โหมด GitHub Copilot Agent บนเซิร์ฟเวอร์ใน Visual Studio Code** ที่นี่ เราจะดูการรัน MCP Server ของเราจากภายใน Visual Studio Code [ไปที่บทเรียน](04-vscode/README.md)

- **5 เซิร์ฟเวอร์การขนส่ง stdio** การขนส่ง stdio เป็นมาตรฐานที่แนะนำสำหรับการสื่อสาร MCP เซิร์ฟเวอร์-ไคลเอนต์ในท้องถิ่น มอบการสื่อสาร subprocess ที่ปลอดภัยพร้อมการแยกกระบวนการในตัว [ไปที่บทเรียน](05-stdio-server/README.md)

- **6 HTTP Streaming กับ MCP (Streamable HTTP)** เรียนรู้เกี่ยวกับการขนส่งสตรีมมิ่ง HTTP สมัยใหม่ (วิธีที่แนะนำสำหรับเซิร์ฟเวอร์ MCP ระยะไกลตาม [MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http)) การแจ้งเตือนความคืบหน้า และวิธีการใช้งานเซิร์ฟเวอร์และไคลเอนต์ MCP แบบเรียลไทม์ที่ปรับขนาดได้โดยใช้ Streamable HTTP [ไปที่บทเรียน](06-http-streaming/README.md)

- **7 การใช้ AI Toolkit สำหรับ VSCode** เพื่อใช้และทดสอบไคลเอนต์และเซิร์ฟเวอร์ MCP ของคุณ [ไปที่บทเรียน](07-aitk/README.md)

- **8 การทดสอบ** ที่นี่เราจะเน้นเป็นพิเศษว่าทำอย่างไรจึงจะทดสอบเซิร์ฟเวอร์และไคลเอนต์ของเราในรูปแบบต่างๆ [ไปที่บทเรียน](08-testing/README.md)

- **9 การปรับใช้** บทนี้จะดูวิธีต่าง ๆ ในการปรับใช้โซลูชัน MCP ของคุณ [ไปที่บทเรียน](09-deployment/README.md)

- **10 การใช้งานเซิร์ฟเวอร์ขั้นสูง** บทนี้ครอบคลุมการใช้งานเซิร์ฟเวอร์ขั้นสูง [ไปที่บทเรียน](./10-advanced/README.md)

- **11 การตรวจสอบสิทธิ์** บทนี้ครอบคลุมวิธีการเพิ่มการตรวจสอบสิทธิ์อย่างง่าย ตั้งแต่ Basic Auth ไปจนถึงการใช้ JWT และ RBAC แนะนำให้เริ่มที่นี่ก่อน แล้วจึงดูหัวข้อขั้นสูงในบทที่ 5 และดำเนินการเสริมความปลอดภัยเพิ่มเติมตามคำแนะนำในบทที่ 2 [ไปที่บทเรียน](./11-simple-auth/README.md)

- **12 โฮสต์ MCP** ตั้งค่าและใช้ไคลเอนต์โฮสต์ MCP ยอดนิยม เช่น Claude Desktop, Cursor, Cline, และ Windsurf เรียนรู้ประเภทของการขนส่งและวิธีแก้ไขปัญหา [ไปที่บทเรียน](./12-mcp-hosts/README.md)

- **13 MCP Inspector** แก้ไขข้อบกพร่องและทดสอบเซิร์ฟเวอร์ MCP ของคุณแบบโต้ตอบโดยใช้เครื่องมือ MCP Inspector เรียนรู้การแก้ไขปัญหาเครื่องมือ ทรัพยากร และข้อความโปรโตคอล [ไปที่บทเรียน](./13-mcp-inspector/README.md)

- **14 การสุ่มตัวอย่าง** สร้างเซิร์ฟเวอร์ MCP ที่ทำงานร่วมกับไคลเอนต์ MCP ในงานที่เกี่ยวข้องกับ LLM (เลิกใช้ในตัวอย่างการปล่อยเวอร์ชัน `2026-07-28` แต่ยังใช้ได้กับ `2025-11-25`) [ไปที่บทเรียน](./14-sampling/README.md)

- **15 แอป MCP** สร้างเซิร์ฟเวอร์ MCP ที่ตอบกลับด้วยคำแนะนำ UI ด้วย [ไปที่บทเรียน](./15-mcp-apps/README.md)

โปรโตคอล Model Context (MCP) เป็นโปรโตคอลเปิดที่มาตรฐานวิธีการที่แอปพลิเคชันให้บริบทแก่ LLM คิดว่า MCP เหมือนพอร์ต USB-C สำหรับแอป AI - มันให้วิธีการมาตรฐานในการเชื่อมต่อโมเดล AI กับแหล่งข้อมูลและเครื่องมือที่แตกต่างกัน

## วัตถุประสงค์การเรียนรู้

เมื่อสิ้นสุดบทเรียนนี้ คุณจะสามารถ:

- ตั้งค่าสภาพแวดล้อมการพัฒนาสำหรับ MCP ใน C#, Java, Python, TypeScript และ JavaScript
- สร้างและปรับใช้เซิร์ฟเวอร์ MCP เบื้องต้นพร้อมฟีเจอร์เฉพาะ (ทรัพยากร, คำกระตุ้น, และเครื่องมือ)
- สร้างแอปโฮสต์ที่เชื่อมต่อกับเซิร์ฟเวอร์ MCP
- ทดสอบและแก้ไขปัญหาการใช้งาน MCP
- เข้าใจความท้าทายทั่วไปในการตั้งค่าและแนวทางแก้ไข
- เชื่อมต่อการใช้งาน MCP ของคุณกับบริการ LLM ยอดนิยม

## การตั้งค่าสภาพแวดล้อม MCP ของคุณ

ก่อนที่คุณจะเริ่มทำงานกับ MCP การเตรียมสภาพแวดล้อมการพัฒนาและเข้าใจขั้นตอนการทำงานพื้นฐานเป็นสิ่งสำคัญ ส่วนนี้จะชี้นำคุณผ่านขั้นตอนการตั้งค่าเริ่มต้นเพื่อให้การเริ่มต้นกับ MCP เป็นไปอย่างราบรื่น

### ข้อกำหนดเบื้องต้น

ก่อนที่จะลงมือพัฒนา MCP ให้แน่ใจว่าคุณมี:

- **สภาพแวดล้อมการพัฒนา**: สำหรับภาษาที่คุณเลือกใช้ (C#, Java, Python, TypeScript หรือ JavaScript)
- **IDE/Editor**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm หรือโปรแกรมแก้ไขโค้ดสมัยใหม่อื่นๆ
- **ตัวจัดการแพ็กเกจ**: NuGet, Maven/Gradle, pip หรือ npm/yarn
- **กุญแจ API**: สำหรับบริการ AI ใดๆ ที่คุณวางแผนจะใช้ในแอปโฮสต์ของคุณ


### SDK อย่างเป็นทางการ

ในบทต่อไปนี้ คุณจะเห็นโซลูชันที่สร้างด้วย Python, TypeScript, Java และ .NET นี่คือ SDK อย่างเป็นทางการทั้งหมดที่ได้รับการสนับสนุน

MCP มี SDK อย่างเป็นทางการสำหรับหลายภาษา (สอดคล้องกับ [MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/)):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - ดูแลร่วมกับ Microsoft
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - ดูแลร่วมกับ Spring AI
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - การใช้งานอย่างเป็นทางการของ TypeScript
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - การใช้งานอย่างเป็นทางการของ Python (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - การใช้งานอย่างเป็นทางการของ Kotlin
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - ดูแลร่วมกับ Loopwork AI
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - การใช้งานอย่างเป็นทางการของ Rust
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - การใช้งานอย่างเป็นทางการของ Go

## ข้อสรุปสำคัญ

- การตั้งค่าสภาพแวดล้อมการพัฒนา MCP เป็นเรื่องง่ายด้วย SDK ที่เฉพาะเจาะจงสำหรับแต่ละภาษา
- การสร้างเซิร์ฟเวอร์ MCP เกี่ยวข้องกับการสร้างและลงทะเบียนเครื่องมือพร้อมสคีมาที่ชัดเจน
- ไคลเอนต์ MCP เชื่อมต่อกับเซิร์ฟเวอร์และโมเดลเพื่อใช้ประโยชน์จากความสามารถที่ขยายออกไป
- การทดสอบและแก้ไขข้อผิดพลาดเป็นสิ่งจำเป็นสำหรับการใช้งาน MCP ที่น่าเชื่อถือ
- ตัวเลือกการปรับใช้งานมีตั้งแต่การพัฒนาในเครื่องจนถึงโซลูชันบนคลาวด์
