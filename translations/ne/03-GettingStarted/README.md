## सुरु गर्ने तरिका  

[![Build Your First MCP Server](../../../translated_images/ne/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(उपरोक्त छवि क्लिक गरी यस पाठको भिडियो हेर्नुहोस्)_

यो खण्डमा विभिन्न पाठहरू समावेश छन्:

- **1 तपाईंको पहिलो सर्भर**, यस पहिलो पाठमा, तपाईंले आफ्नो पहिलो सर्भर कसरी बनाउन र इन्स्पेक्टर उपकरणको साथ निरीक्षण गर्ने तरिका सिक्नुहुनेछ, जुन तपाईंको सर्भर परीक्षण र डिबग गर्ने मूल्यवान तरिका हो, [पाठमा जानुहोस्](01-first-server/README.md)

- **2 क्लाइन्ट**, यस पाठमा, तपाईंले कसरी क्लाइन्ट लेख्ने जसले तपाईंको सर्भरमा जडान गर्न सक्छ भन्ने सिक्नुहुनेछ, [पाठमा जानुहोस्](02-client/README.md)

- **3 LLM सहित क्लाइन्ट**, क्लाइन्ट लेख्ने अझ राम्रो तरिका भनेको LLM थप्नु हो ताकि यो तपाईंको सर्भरसँग "सम्झौता" गर्न सक्दछ कि के गर्ने, [पाठमा जानुहोस्](03-llm-client/README.md)

- **4 GitHub Copilot Agent मोड प्रयोग गरी Visual Studio Code मा सर्भर उपयोग**। यहाँ, हामी Visual Studio Code भित्रबाट हाम्रो MCP सर्भर चलाउने कुरा हेर्दैछौं, [पाठमा जानुहोस्](04-vscode/README.md)

- **5 stdio ट्रान्सपोर्ट सर्भर**, stdio ट्रान्सपोर्ट स्थानीय MCP सर्भर-देखि-क्लाइन्ट सञ्चारको लागि सिफारिस गरिएको मानक हो, जसले सुरक्षित subprocess-आधारित सञ्चार र इनबिल्ट प्रक्रिया पृथकीकरण प्रदान गर्दछ [पाठमा जानुहोस्](05-stdio-server/README.md)

- **6 MCPसँग HTTP स्ट्रिमिङ (Streamable HTTP)**। आधुनिक HTTP स्ट्रिमिङ ट्रान्सपोर्ट (MCP स्पेसिफिकेसन 2025-11-25 अनुसार रिमोट MCP सर्भरहरूको लागि सिफारिस गरिएको तरिका), प्रगति सूचनाहरू, र कसरी स्केलेबल, वास्तविक-समय MCP सर्भर र क्लाइन्टहरू Streamable HTTP प्रयोग गरी लागू गर्ने सिक्नुहोस्। [पाठमा जानुहोस्](06-http-streaming/README.md)

- **7 VSCode को लागि AI उपकरण किट प्रयोग गर्ने** ताकि तपाईंको MCP क्लाइन्ट र सर्भरहरू प्रयोग र परीक्षण गर्न सकियोस् [पाठमा जानुहोस्](07-aitk/README.md)

- **8 परीक्षण**। यहाँ हामी विशेष गरी कसरी विभिन्न तरिकाले हाम्रो सर्भर र क्लाइन्ट परीक्षण गर्न सकिन्छ भन्नेमा ध्यान दिनेछौं, [पाठमा जानुहोस्](08-testing/README.md)

- **9 डिप्लोइमेन्ट**। यस अध्यायले तपाईंको MCP समाधानहरू डिप्लोइ गर्न विभिन्न तरिकाहरूलाई हेर्नेछ, [पाठमा जानुहोस्](09-deployment/README.md)

- **10 उन्नत सर्भर प्रयोग**। यस अध्यायले उन्नत सर्भर प्रयोग कभर गर्दछ, [पाठमा जानुहोस्](./10-advanced/README.md)

- **11 प्रमाणीकरण**। यस अध्यायले सरल प्रमाणीकरण कसरी थप्ने कभर गर्दछ, Basic Auth देखि JWT र RBAC सम्म। तपाईंलाई यहाँबाट सुरु गरेर अध्याय 5 मा उन्नत विषयहरू हेर्न र अध्याय 2 मा सिफारिस गरेको थप सुरक्षा कडा गर्ने प्रोत्साहन दिइन्छ, [पाठमा जानुहोस्](./11-simple-auth/README.md)

- **12 MCP होस्टहरू**। लोकप्रिय MCP होस्ट क्लाइन्टहरू जस्तै Claude Desktop, Cursor, Cline, र Windsurf कन्फिगर र प्रयोग गर्नुहोस्। ट्रान्सपोर्ट प्रकारहरू र समस्या समाधान सिक्नुहोस्, [पाठमा जानुहोस्](./12-mcp-hosts/README.md)

- **13 MCP इन्स्पेक्टर**। MCP इन्स्पेक्टर उपकरण प्रयोग गरी अन्तर्क्रियात्मक रूपमा तपाईंका MCP सर्भरहरू डिबग र परीक्षण गर्नुहोस्। उपकरण, स्रोतहरू र प्रोटोकल सन्देशहरू समस्या समाधान सिक्नुहोस्, [पाठमा जानुहोस्](./13-mcp-inspector/README.md)

- **14 नमूना बनाउने (Sampling)**। MCP क्लाइन्टहरूसँग LLM सम्बन्धित कार्यहरूमा सहकार्य गर्ने MCP सर्भरहरू बनाउनुहोस् (विधिवत रूपमा `2026-07-28` रिलिज क्यान्डिडेटमा अवमूल्यन गरिएको छ; तर `2025-11-25` को लागि वैध छ)। [पाठमा जानुहोस्](./14-sampling/README.md)

- **15 MCP एपहरू**। UI निर्देशनहरूसँग प्रतिक्रिया दिने MCP सर्भरहरू निर्माण गर्नुहोस्, [पाठमा जानुहोस्](./15-mcp-apps/README.md)

मोडेल कन्टेक्स्ट प्रोटोकल (MCP) एउटा खुल्ला प्रोटोकल हो जसले अनुप्रयोगहरूले LLMs लाई सन्दर्भ कसरी प्रदान गर्ने भन्ने मानकीकरण गर्दछ। MCP लाई AI अनुप्रयोगहरूको लागि USB-C पोर्ट जस्तो सोँच्नुहोस् - यसले AI मोडेलहरूलाई विभिन्न डेटा स्रोत र उपकरणहरूले जडान गर्ने मानकीकृत तरिका प्रदान गर्दछ।

## सिकाइ उद्देश्यहरू

यस पाठ अन्त्यमा, तपाईं सक्नुहुनेछ:

- C#, Java, Python, TypeScript, र JavaScript मा MCP को विकास वातावरण सेटअप गर्ने
- साधारण MCP सर्भरहरू कस्टम सुविधाहरू (स्रोतहरू, प्रम्प्टहरू, र उपकरणहरू) सहित निर्माण र डिप्लोइ गर्ने
- MCP सर्भरहरूमा जडान हुने होस्ट अनुप्रयोगहरू सिर्जना गर्ने
- MCP कार्यान्वयनहरू परीक्षण र डिबग गर्ने
- साझा सेटअप चुनौतीहरू र तिनीहरूको समाधान बुझ्ने
- तपाईंको MCP कार्यान्वयनहरू लोकप्रिय LLM सेवाहरूमा जडान गर्ने

## तपाईंको MCP वातावरण सेटअप गर्ने तरिका

MCP सँग काम गर्न सुरु गर्नु अघि, विकास वातावरण तयार पार्नु र आधारभूत कार्यप्रवाह बुझ्नु महत्त्वपूर्ण छ। यस खण्डले तपाईंलाई सुरु गर्न आवश्यक प्रारम्भिक कदमहरूको मार्गदर्शन गर्नेछ।

### आवश्यकताहरू

MCP विकासमा डुब्ने अघि, सुनिश्चित गर्नुहोस् तपाईंले:

- **विकास वातावरण**: तपाईंको रोजाइको भाषा (C#, Java, Python, TypeScript, वा JavaScript) को लागि
- **IDE/सम्पादक**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm, वा कुनै आधुनिक कोड सम्पादक
- **प्याकेज म्यानेजर**: NuGet, Maven/Gradle, pip, वा npm/yarn
- **API कुञ्जीहरू**: तपाईंले आफ्नो होस्ट अनुप्रयोगहरूमा प्रयोग गर्ने योजना बनाउनु भएका कुनैपनि AI सेवाहरूका लागि


### आधिकारिक SDK हरू

आगामी अध्यायहरूमा तपाईंले Python, TypeScript, Java र .NET प्रयोग गरेर बनाइएका समाधानहरू देख्नुहुनेछ। यहाँ सबै आधिकारिक रूपमा समर्थन गरिएका SDK हरू छन्।

MCP ले धेरै भाषाको लागि आधिकारिक SDK हरू प्रदान गर्दछ ([MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/) सँग मिल्दोजुल्दो):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - Microsoft सँग सहकार्यमा मर्मत गरिएको
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI सँग सहकार्यमा मर्मत गरिएको
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - आधिकारिक TypeScript कार्यान्वयन
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - आधिकारिक Python कार्यान्वयन (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - आधिकारिक Kotlin कार्यान्वयन
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI सँग सहकार्यमा मर्मत गरिएको
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - आधिकारिक Rust कार्यान्वयन
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - आधिकारिक Go कार्यान्वयन

## मुख्य बुँदाहरू

- MCP विकास वातावरण भाषा-विशिष्ट SDK हरूको साथ सजिलो सेटअप हुन्छ
- MCP सर्भरहरू निर्माण गर्दा स्पष्ट योजना सहित उपकरणहरू सिर्जना र दर्ता गर्नुपर्ने हुन्छ
- MCP क्लाइन्टहरूले सर्भर र मोडेलहरूमा जडान गर्दै बढाइएको क्षमता प्रयोग गर्छन्
- परीक्षण र डिबगिंग विश्वसनीय MCP कार्यान्वयनको लागि आवश्यक छ
- डिप्लोइमेन्ट विकल्पहरू स्थानीय विकासदेखि क्लाउड-आधारित समाधानसम्म फरक छन्
